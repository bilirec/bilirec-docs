[bilirec-web](https://github.com/bilirec/bilirec-web) 是 bilirec 的通用 Web 管理界面，线上地址为 [app.bilirec.org](https://app.bilirec.org/)。它是一个独立的前端项目，通过 REST API 连接任意 bilirec 后端实例。

面向最终用户的使用说明见 [Web 界面（PWA）](/zh-cn/guides/web-ui/)。API 端点详见 [开发者集成](/zh-cn/api/overview/)。

## 项目定位

- **主要目标**：为 bilirec 后端提供移动优先的 PWA 管理界面。
- **技术方案**：React SPA + Service Worker（PWA）+ Axios 调用后端 REST API。
- **部署方式**：静态资源托管（官方实例部署于 Cloudflare Workers）；后端须单独运行。
- **多后端支持**：登录页可配置服务器地址，地址保存在 `localStorage`（键名 `server-url`）。

<Aside type="note">
线上 `app.bilirec.org` 为 **HTTPS**。浏览器混合内容策略下，从该页面只能连接本机 HTTP 后端（`localhost` / `127.0.0.1`）；远程 HTTP 局域网地址会被拦截，须使用 HTTPS 隧道（如 [FRP](/zh-cn/guides/frp/)）。详见 [Web 界面](/zh-cn/guides/web-ui/#连接自架后端)。
</Aside>

## 功能概览

| 功能 | 说明 |
|------|------|
| 录制管理 | 启动/停止录制，查看 `recording` / `recovering` / `idle` 状态 |
| 直播订阅 | 订阅房间、管理自动录制与通知设置 |
| 网页推送 | 通过 Service Worker 接收开播等 Web Push 通知 |
| 文件浏览 | 浏览录制文件，支持流式下载（File System Access API → StreamSaver 回退） |
| MP4 预览 | 应用内预览 MP4 录制 |
| 文件共享 | 生成带过期时间的分享链接 |
| 格式转换 | 提交与管理 FLV/TS/FMP4 ↔ MP4 转换任务 |
| 磁盘监测 | 实时磁盘占用与分项统计 |
| 角色控制 | 管理员 / 查看者（查看者为只读界面） |
| 深色模式 | 浅色/深色主题，支持跟随系统 |
| 多语言 | 简体中文 / 繁体中文（i18next） |

## 技术栈

| 组件 | 用途 |
|------|------|
| React 19 + TypeScript | UI 框架 |
| Vite 7 | 构建工具 |

## 本地开发

### 前置条件

- Node.js 与 **pnpm**
- 一个正在运行的 bilirec 后端（本地或远程 HTTPS）

### 启动项目

```bash
pnpm install
pnpm dev
```

开发服务器默认监听 `http://localhost:3000`（`vite.config.ts` 中 `server.port = 3000`，`host: true` 允许局域网访问开发服务器本身）。

在登录页填入后端地址。本地开发时若后端跑在本机，使用 `http://localhost:8080` 或 `http://127.0.0.1:8080`。

### 构建与预览

```bash
pnpm build    # tsc + vite build，输出 dist/
pnpm preview  # 本地预览生产构建
```

## 自行部署（Cloudflare）

官方 [app.bilirec.org](https://app.bilirec.org/) 通过 Cloudflare Workers 托管 `dist/` 静态资源（见 `wrangler.jsonc`）。你也可以 fork 仓库后一键部署到自己的 Cloudflare 账号：

[![Deploy to Cloudflare](https://deploy.workers.cloudflare.com/button)](https://deploy.workers.cloudflare.com/?url=https://github.com/bilirec/bilirec-web)

<Aside type="caution" title="部署注意事项">

| 事项 | 说明 |
|------|------|
| 后端 `FRONTEND_URL` | 部署完成后，将 bilirec 后端的 `FRONTEND_URL` 设为你的前端 HTTPS 域名（CORS 与 Cookie 认证）。见 [服务器与网络配置](/zh-cn/configuration/server-network/) |
| 自架前端 + 不同根域名后端 | 若前端域名与后端（如 FRP 的 `*.tunnel.bilirec.org`）**根域名不同**，除 `PRODUCTION_MODE=true` 外，可能还需在浏览器允许第三方 Cookie。官方 `app.bilirec.org` 连 FRP 一般无此问题。见 [FRP — 根域名是否相同](/zh-cn/guides/frp/#根域名是否相同) |
| 须使用 HTTPS | PWA 安装与 Web Push 需要 HTTPS；自架 HTTPS 前端可连接局域网 HTTP 后端（官方 `app.bilirec.org` 仅允许连接本机 `localhost` / `127.0.0.1`） |
| 与官方前端取舍 | `app.bilirec.org` 适合本机、Android 内嵌后端或 FRP HTTPS 隧道；自架前端适合自定义域名或需从 HTTPS 页面连接局域网 HTTP 后端的场景 |
| Fork 行为 | 按钮会在你的 GitHub 创建仓库 fork 并绑定 Workers Builds，后续 push 自动部署 |
| 后端须单独运行 | 前端仅为静态资源，bilirec 录制后端须另行[安装与启动](/zh-cn/guides/installation/) |
| 首次连接 | 打开自架前端 → 填入 bilirec 后端地址 → 必要时登录 → 完成 B 站扫码。见 [快速开始](/zh-cn/guides/quick-start/) |

</Aside>

### 手动部署

自行部署时也可不使用 Cloudflare 按钮：

1. 执行 `pnpm build` 生成 `dist/`。
2. 将 `dist/` 部署到任意静态托管（Cloudflare Pages、Nginx、对象存储 + CDN 等），或使用 `pnpm deploy`（`wrangler deploy`）。
3. **须使用 HTTPS**，否则 PWA 安装与 Web Push 等功能受限。
4. 确保后端 `FRONTEND_URL` 包含你的前端域名（CORS）。见 [服务器与网络配置](/zh-cn/configuration/server-network/)。

## 认证与后端连接

- 登录时 `apiClient.setBaseURL(url)` 设置后端地址，并写入 `localStorage` 的 `server-url`。
- 若提供 `USERNAME` / `PASSWORD`，调用 `POST /login`，JWT 通过 **HttpOnly Cookie** 返回，不在 JavaScript 中存储令牌。
- 未配置账号密码时，前端会尝试无认证访问；后端返回 401 则提示需要登录。

## PWA 与 Service Worker

- 构建策略：`vite-plugin-pwa` 的 **injectManifest**，入口为 `src/sw.ts`。
- `registerType: 'prompt'` — 由浏览器决定是否显示安装提示。
- Service Worker 处理 Web Push 事件，并从 IndexedDB 读取 `server-url` 以定位推送订阅端点。
- 用户安装说明见 [Web 界面 — 安装 PWA](/zh-cn/guides/web-ui/#安装-pwa)。

<Aside type="tip">
修改 Service Worker 后须重新 `pnpm build` 并确保用户刷新或更新 PWA，新 SW 才会生效。
</Aside>

## 实时刷新

主界面每 **5 秒**自动轮询后端状态；页面不可见（标签页隐藏）时暂停轮询，以节省资源。
