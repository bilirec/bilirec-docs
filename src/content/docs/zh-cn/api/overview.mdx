bilirec 提供完整的 RESTful API，供程序调用、脚本控制或自行编写管理界面。在 Android 上嵌入后端时，可先通过 [Android 库接口](/zh-cn/development/android-library/) 启动服务，再调用本文档中的 HTTP API（或直接使用 [bilirec-web](https://app.bilirec.org/) 等现成前端）。

## Swagger UI

服务器运行时，在根路径 `/` 提供 Swagger UI — 在浏览器中打开 `http://your-host:8080/` 即可查看与测试所有 API。

## 认证

若设置了 `USERNAME` 与 `PASSWORD`，API 启用基于 JWT 的认证。详见 [认证 API](/zh-cn/api/auth/)。

未设置用户名/密码时，API 默认为公开访问。

## 通用约定

- 基础 URL：`http://host:port`（默认 `http://localhost:8080`）
- 需要认证时，先 `POST /login` 获取 `jwtToken` cookie，后续请求携带该 cookie
- 错误响应通常为 JSON：`{"error": "描述信息"}`
