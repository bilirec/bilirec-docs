## 开始录制

```http
POST /record/:roomID/start?duration_minutes=<N>&stream_profile=<profile>&qn=<quality>&only_audio=<bool>
```

### duration_minutes

| 值 | 说明 |
| -- | ---- |
| 不传 / `0` | 使用系统预设（`MAX_RECORDING_HOURS`） |
| `-1` | 无限录制 |
| 正整数 | N 分钟后自动停止 |

### stream_profile

| 值 | 说明 |
| -- | ---- |
| 不传 | 自动选择（依次尝试 HTTP-FLV → HLS-fMP4 → HLS-TS） |
| `http-flv` | HTTP-FLV |
| `hls-ts` | HLS-TS |
| `hls-fmp4` | HLS-fMP4 |

### qn（画质码）

| 值 | 画质 |
| -- | ---- |
| 不传 | 原画（`10000`） |
| `80` | 流畅 |
| `150` | 高清 |
| `250` | 超清 |
| `400` | 蓝光 |
| `10000` | 原画 |
| `20000` | 4K |
| `30000` | 杜比 |

### only_audio

| 值 | 说明 |
| -- | ---- |
| 不传 / `false` | 录制音视频 |
| `true` | 仅录制音频 |

**注意：**

- 录 4K / 杜比请显式传 `qn`（如 `qn=20000`、`qn=30000`）
- `qn ≥ 20000` 时自动启用 `*_HIGH` 高画质内存池，一般无需改环境变量
- 若遇卡顿或内存不足，参见 [高码率 / 4K 调参](/zh-cn/configuration/high-bitrate-tuning/) 与 [内存占用估算](/zh-cn/configuration/memory-estimation/)

```bash
curl -X POST "http://127.0.0.1:8080/record/12345/start?duration_minutes=30&stream_profile=http-flv&qn=150&only_audio=true"
```

### 错误响应

| HTTP | 说明 |
| ---- | ---- |
| `400` | 无效参数、房间未开播、已在录制、正在恢复流等 |
| `409` | 该房间录制正在启动中（并发请求冲突） |
| `410` | 无法连接到任何可用流 URL |
| `429` | 已达到 `MAX_CONCURRENT_RECORDINGS` 上限 |
| `507` | 磁盘可用空间低于 `MIN_DISK_SPACE_BYTES` |

常见 `400` 消息：`该房间已在录制中`、`该房间正在恢复流`、`房间并非直播状态`、`无可用的视频流 URL`。

## 停止录制

```http
POST /record/:roomID/stop
```

## 批量获取状态

```http
POST /record/statuses
Content-Type: application/json

{ "roomIDs": [123, 456, 789] }
```

也支持 query：`?roomIDs=123,456,789`

```json
{ "123": "recording", "456": "idle", "789": "recovering" }
```

状态值：`idle`（未录制）、`recording`（录制中）、`recovering`（流中断，正在重连恢复）。

## 批量获取统计

```http
POST /record/stats
Content-Type: application/json

{ "roomIDs": [123, 456, 789] }
```

```json
{
  "123": {
    "bytes_written": 1048576,
    "status": "recording",
    "start_time": 1234567890,
    "elapsed_seconds": 120,
    "output_path": "records/tester-123/live-title.flv"
  }
}
```

## 列出所有任务

```http
GET /record/list
```
