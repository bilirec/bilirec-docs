SSD 随机读写很快，不必像 microSD 那样担心小块写入磨损；默认的「多路轮流写盘」反而会限制 SSD 并发优势。

## 推荐配置

```bash
export LIVE_STREAM_WRITER_SYNC_PERIOD_SECS=30
export LIVE_STREAM_WRITER_COLD_CACHE_RELEASE_SECS=0
export DROP_FILE_PAGE_CACHE=false
export SKIP_SMALL_FLUSH=false
export SEQUENTIAL_WRITE=false
export LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE=128
export MAX_CONCURRENT_RECORDINGS=10
```

| 参数 | 调整 | 你会注意到什么 |
|------|------|----------------|
| `LIVE_STREAM_WRITER_SYNC_PERIOD_SECS=30` | 录制中定期整文件落盘 | 断电少丢数据；仅当 `COLD_CACHE_RELEASE_SECS>0` 时 fsync 后才会顺带释放冷前缀 |
| `LIVE_STREAM_WRITER_COLD_CACHE_RELEASE_SECS=0` | 关闭录制中清理文件缓存 | 监控数字可能略高，但录完立刻播放更快 |
| `DROP_FILE_PAGE_CACHE=false` | 关档时不释放文件缓存 | 录完立刻播放更快 |
| `SKIP_SMALL_FLUSH=false` | 不跳过极短直播保护 | 更早上盘，文件更完整 |
| `SEQUENTIAL_WRITE=false` | 允许多路同时写盘 | SSD 多路并发更顺畅 |
| `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE=128` | 加大写盘前排队 | 更能扛网络突发，内存占用上升 |
| `MAX_CONCURRENT_RECORDINGS=10` | 放宽同时录制路数 | 视 CPU 与内存调整 |

<Aside type="tip">
若底层使用 ZFS 等自带大块缓存的文件系统，可把 `LIVE_STREAM_WRITER_SYNC_PERIOD_SECS` 设为 `0` 或 `60`，减少应用层反复落盘。
</Aside>
