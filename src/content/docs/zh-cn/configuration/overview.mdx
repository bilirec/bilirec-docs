所有配置通过**环境变量**设置。bilirec 支持多种配置来源，按以下优先级加载：

## 加载顺序

1. **`.env.local`** — 当前工作目录下（若存在，最高优先级）
2. **`.env`** — 可执行文件同目录（首次运行自动生成模板）
3. **环境变量** — 覆盖文件中的值
4. **Docker 环境** — 检测到 `/.dockerenv` 时跳过 `.env` 加载，仅使用环境变量

## 修改配置

**二进制模式：**

编辑 `.env.local` 或 `.env` 文件，或直接设置环境变量：

**Linux / macOS：**

```bash
export MAX_CONCURRENT_RECORDINGS=5
./bilirec-linux-amd64
```

**Windows PowerShell：**

```powershell
$env:MAX_CONCURRENT_RECORDINGS="5"
.\bilirec-windows.exe
```

**Docker 模式：**

通过 `-e` 传入环境变量：

```bash
docker run -e MAX_CONCURRENT_RECORDINGS=5 ...
```

## 重要说明

- `PUBLIC_BASE_URL` 为空或无效时，预签名 URL 会回退为相对地址；CloudConvert 在该情况下会拒绝创建任务
- Web Push 的 VAPID key 在启动时自动生成并写入 `SECRET_DIR`，后续重启会复用已有 key
