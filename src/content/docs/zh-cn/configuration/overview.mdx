所有配置通过**环境变量**设置。bilirec 的加载逻辑与 README 保持一致：

## 加载顺序

1. 先尝试读取当前工作目录的 **`.env.local`**（若存在）
2. 若不存在 `.env.local`，则读取（或生成）可执行文件同目录的 **`.env`**
3. 在 **Docker 环境**（检测到 `/.dockerenv`）下，跳过 `.env` 自动加载，仅使用容器环境变量
4. 你也可以直接设置系统环境变量，覆盖 `.env.local` / `.env` 中的值

## 修改配置

**二进制模式：**

编辑 `.env.local` 或 `.env` 文件，或直接设置环境变量：

**Linux / macOS：**

```bash
export MAX_CONCURRENT_RECORDINGS=5
./bilirec-linux-amd64
```

**Windows PowerShell：**

```powershell
$env:MAX_CONCURRENT_RECORDINGS="5"
.\bilirec-windows.exe
```

**Docker 模式：**

通过 `-e` 传入环境变量：

```bash
docker run -e MAX_CONCURRENT_RECORDINGS=5 ...
```

## 重要说明

- `PUBLIC_BASE_URL` 为空或无效时，预签名 URL 会回退为相对地址；CloudConvert 在该情况下会拒绝创建任务
- Web Push 的 VAPID key 在启动时自动生成并写入 `SECRET_DIR`，后续重启会复用已有 key
