Android 模式（`libbilirec.so`）会先注入一组移动端默认值，再套用 `env` 覆盖。

## 默认行为

以下配置是 Android 模式下的默认行为（可通过 `env` 覆盖）：

| 变量 | 默认值 | 原因 |
|------|--------|------|
| `BILIBILI_LOGIN_MODE` | `controller` | 通过 Web UI 扫码登录 |
| `FRP_ENABLED` | `false` | 手机无需内网穿透 |
| `SILENT_ACCESS_LOG` | `true` | 减少日志输出 |
| `SKIP_SMALL_FLUSH` | `false` | 优先保障录制完整性，不延迟建文件 |
| `DROP_FILE_PAGE_CACHE` | `true` | 关档/转码后释放整文件页缓存；录製中另由 `LIVE_STREAM_WRITER_COLD_CACHE_RELEASE_SECS` 处理冷前缀 |
| `SEQUENTIAL_WRITE` | `false` | 手机内置存储随机 I/O 较好，默认不序列化写入 |

[bilirec-mobile](https://github.com/bilirec/bilirec-mobile) 在 **服务启动前设置 → 存储策略 → 序列化写入** 中可切换 `SEQUENTIAL_WRITE`。若输出路径选择 microSD 卡，建议开启此项。也可在 `Start` 的 `env` 中传入 `SEQUENTIAL_WRITE=true` 或 `false` 覆盖。

## App 设置项对照

bilirec-mobile 将用户友好的选项映射为后端环境变量。服务启动时，App 会合并 **managed**（策略页开关/滑块）与 **developer**（开发者选项中的自定义键值）环境变量；`OUTPUT_DIR` 由输出路径单独传入。开发者选项中的同名键会覆盖策略页设置。

| App 设置（简体中文界面） | 环境变量 | 可选值 / 默认 |
|------------------------|---------|--------------|
| 输出路径 | `OUTPUT_DIR` | 未设置时用 `{basePath}/records` |
| 序列化写入 | `SEQUENTIAL_WRITE` | `true` / `false` |
| 单次录制时长上限 | `MAX_RECORDING_HOURS` | `0`～`12`（`0`=不限） |
| 启动前最低可用空间 | `MIN_DISK_SPACE_BYTES` | `2` / `5` / `10` GB |
| 下线后服务重试时长 | `MAX_RETRY_MINUTES` | `5`～`30` 分钟 |
| 同时录制上限 | `MAX_CONCURRENT_RECORDINGS` | `3`～`6` |
| 录完自动转为 MP4 | `CONVERT_TO_MP4` | `true` / `false` |
| 转档后删除原始文件 | `DELETE_SOURCE_AFTER_CONVERT` | `true` / `false` |
| 录制中允许转换 | `FFMPEG_ALLOW_DURING_RECORDING` | `true` / `false` |
| 允许转档的「最高录制路数」 | `FFMPEG_ALLOW_DURING_RECORDING_MAX_ACTIVE_RECORDINGS` | `0`～`5`（`0`=不限） |
| 本地通知模式 | `NOTIFY_SSE_TOKEN` | 开启时由 App 自动生成 |

界面操作说明见 [Android App](/zh-cn/guides/android/)。

## 自动注入

| 变量 | 默认值 |
|------|--------|
| `HOST` | `127.0.0.1` |
| `PORT` | `8080` |
| `OUTPUT_DIR` | `{basePath}/records` |
| `SECRET_DIR` | `{basePath}/secrets` |
| `DATABASE_DIR` | `{basePath}/database` |
| `FRONTEND_URL` | `https://app.bilirec.org` |

## I/O 与磁盘

与服务器版相比，Android 使用更保守的缓冲与更低的磁盘空间阈值，避免前台卡顿：

| 变量 | Android 默认 | 服务器默认 |
|------|-------------|-----------|
| `MIN_DISK_SPACE_BYTES` | `2147483648` (2 GB) | `5368709120` (5 GB) |
| `LIVE_STREAM_WRITER_BUFFER_SIZE` | `4194304` (4 MB) | `8388608` (8 MB) |
| `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE` | `32` | `64` |
| `READ_STREAM_CHAN_BUFFER_SIZE_HIGH` | `32` | `48` |
| `LIVE_STREAM_WRITER_SYNC_PERIOD_SECS` | `0` | `0` |
| `LIVE_STREAM_WRITER_COLD_CACHE_RELEASE_SECS` | `60` | `60` |
| `LIVE_STREAM_WRITER_FLUSH_PERIOD_SECS` | `15` | `15` |
| `DROP_FILE_PAGE_CACHE` | `true` | `true` |

高画质池（`READ_STREAM_BYTES_POOL_SIZE_HIGH`、`LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH`）默认均为 1 MB，与服务器版一致。

`DROP_FILE_PAGE_CACHE` 开启后，下次播放同一文件可能需重新从磁盘读取；若介意可设 `false`，详见 [常见问题](/zh-cn/guides/faq/#drop_file_page_cache-什么时候用代价是什么)。

## 动态资源限制

启动时会根据 `MAX_CONCURRENT_RECORDINGS`（默认 `3`）自动调整：

- **内存上限**：`60 MB + 36 MB × 最大录制路数`（通过 Go `GOMEMLIMIT` 软限制）
- **CPU 核心数**：`GOMAXPROCS` 按录制路数与设备物理核心数动态计算（至少 2 核，至多物理核心数的一半）

集成方法见 [Android 库接口](/zh-cn/guides/android-library/)，使用流程见 [Android App](/zh-cn/guides/android/)。
