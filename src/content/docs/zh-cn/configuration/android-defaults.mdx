---
title: Android 默认配置
description: Android 模式与服务器版的默认差异
---

Android 模式（`libbilirec.so`）启动时会先注入一组移动端默认值，再由 App 设置或 `env` 覆盖。

App 里每一项怎么设、对应哪些环境变量，见 [Android App](/zh-cn/guides/android/)。自行集成库见 [Android 库接口](/zh-cn/guides/android-library/)。

## 与服务器版的主要差异

| 项目 | Android 默认 | 服务器默认 | 说明 |
| ---- | ------------ | ---------- | ---- |
| `MIN_DISK_SPACE_BYTES` | 2 GB | 5 GB | 启动前磁盘门槛 |
| `LIVE_STREAM_WRITER_BUFFER_SIZE` | 4 MB | 8 MB | 写盘前合并缓冲更小，省内存 |
| `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE` | 32 | 64 | 写盘排队更浅 |
| `READ_STREAM_CHAN_BUFFER_SIZE_HIGH` | 32 | 48 | 4K 读入排队更浅 |
| `SEQUENTIAL_WRITE` | `false` | `true` | 内置存储随机 I/O 较好；**输出到 microSD 时请在 App 开启序列化写入** |
| `SKIP_SMALL_FLUSH` | `false` | `true` | 优先保证录制完整性，不延迟建文件 |

其余 I/O 与页缓存相关项（`LIVE_STREAM_WRITER_SYNC_PERIOD_SECS=0`、冷释放 60 秒、`DROP_FILE_PAGE_CACHE=true` 等）与服务器相同。CPU / 内存尖峰、缓冲与页缓存怎么调，见 [常见问题 · 性能与容量规划](/zh-cn/guides/faq/#性能与容量规划) 与 [内存占用估算](/zh-cn/configuration/memory-estimation/)。

## 移动端专用默认

| 变量 | 默认值 | 说明 |
| ---- | ------ | ---- |
| `BILIBILI_LOGIN_MODE` | `controller` | 通过 Web UI 扫码登录 |
| `FRP_ENABLED` | `false` | 手机无需内网穿透 |
| `SILENT_ACCESS_LOG` | `true` | 减少访问日志 |

## 启动时自动限制资源

根据 **同时录制上限**（`MAX_CONCURRENT_RECORDINGS`，默认 `3`），启动时会自动设置内存软上限与可用 CPU 核数，一般无需手改：

- **内存软上限** ≈ 60 MB + 每多一路约 36 MB
- **CPU 核数** ≈ 2 核起，每多 3 路加 1 核，不超过设备物理核心数的一半

在 App 里提高 **同时录制上限** 会同步抬高上述限制。若感到发热、卡顿或通知栏内存长期偏高，优先在 App 调低该滑块，再按需查阅 [内存占用估算](/zh-cn/configuration/memory-estimation/)。
