默认配置已覆盖大多数场景：**录 1080p 不用改**；录 4K / 杜比时在 API 里传对应 `qn` 即可，程序会自动用更大块的缓冲。

只有出现卡顿、内存不足、或 HLS 分片特别大时，再按下表微调。

## 高画质自动切换

bilirec 为 **1080p** 与 **高画质（2K/4K/杜比）** 各准备了一套缓冲。录制时根据 `qn` 自动选用，无需在配置里手动开「4K 模式」。

| 画质 | 判定（`qn`） | 读入块 | 读入排队 | 写盘块 |
| ---- | ------------ | ------ | -------- | ------ |
| 1080p 及以下 | `< 20000` | 512 KB | 16 块 | 512 KB |
| 2K / 4K / 杜比 | `≥ 20000` | 1 MB | 48 块 | 1 MB |

环境变量对应关系：`READ_STREAM_BYTES_POOL_SIZE` / `*_HIGH`、`READ_STREAM_CHAN_BUFFER_SIZE` / `*_HIGH`、`LIVE_STREAM_WRITER_BYTES_POOL_SIZE` / `*_HIGH`。

| 要点 | 说明 |
| ---- | ---- |
| 按需占用 | 只有真的在录高画质时才用 `_HIGH` 那套；不录 4K 不会多占内存 |
| HLS | 分片大小由 m3u8 决定；`qn ≥ 20000` 时读入排队深度仍用 `READ_STREAM_CHAN_BUFFER_SIZE_HIGH` |
| 写盘排队 | `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE` 不分档，1080p 与 4K 共用 |

## 常见场景

| 场景 | 要不要改配置 | 建议 |
| ---- | ------------ | ---- |
| 1080p 原画 | 否 | 保持默认 |
| 首次录 4K / 杜比 | 通常否 | API 传 `qn=20000`（或 `30000`）即可 |
| 4K 网络经常卡一下 | 是 | 加大 `READ_STREAM_CHAN_BUFFER_SIZE_HIGH`（默认 48） |
| 4K HLS 分片很大 | 是 | 主要加大 `LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH` |
| 4GB 设备多路 4K | 视情况 | 优先减小 `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE`（如 `48`）或 `MAX_CONCURRENT_RECORDINGS` |

## 推荐起点

三个 `*_HIGH` 默认值通常已够用，下表供参考：

| 场景 | `READ_STREAM_BYTES_POOL_SIZE_HIGH` | `READ_STREAM_CHAN_BUFFER_SIZE_HIGH` | `LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH` | `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE` |
| ---- | ---------------------------------- | ----------------------------------- | ----------------------------------------- | ------------------------------------- |
| 4K / 杜比 FLV 单路 | `1048576` (1 MB) | `48` | `1048576` (1 MB) | `64` |
| 4K / 杜比 HLS 单路 | 对 FLV 读块无影响 | `48` | `1048576` (1 MB) | `64` |
| SSD / NAS 单路极高码率 | `1048576`～`2097152` | `48`～`64` | 与读块 `*_HIGH` 相同 | `64`～`128` |

<Aside type="caution">
调 4K 请改带 `_HIGH` 后缀的变量。不要动不带 `_HIGH` 的基础变量——那是 1080p 用的，改大只会让**所有**录制都多占内存。
</Aside>

## 示例（4K 默认仍不够用时）

```bash
export READ_STREAM_BYTES_POOL_SIZE_HIGH=2097152       # 2 MB，与写侧保持一致
export READ_STREAM_CHAN_BUFFER_SIZE_HIGH=48
export LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH=2097152
export LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE=64       # 多路 4K 可降到 48
```

**内存粗算（单路上限）：** 见 [内存占用估算](/zh-cn/configuration/memory-estimation/)；默认 4K 单路约 165~172 MB。

通过 API 启动高画质录制时，请显式传 `qn` 参数，详见 [录制管理 API](/zh-cn/api/record/)。
