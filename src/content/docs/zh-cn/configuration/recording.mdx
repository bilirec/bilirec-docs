## 并发与时长

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `MAX_CONCURRENT_RECORDINGS` | 最大同时录制数 | `3` |
| `MAX_RECORDING_HOURS` | 单次录制最长时间（小时） | `5` |
| `MAX_RECOVERY_ATTEMPTS` | 单次录制最大重连尝试次数（流仍在线但连接失败时） | `5` |
| `MAX_RETRY_MINUTES` | 直播下线后判断容忍时间（分钟）；下线期间持续重试，超时后结束录制 | `10` |

`MAX_CONCURRENT_RECORDINGS` 直接影响内存峰值。树莓派默认 `3`（1080p 默认缓冲下可验证至 5 路），SSD 环境可放宽至 `10`。规划容器上限请参阅 [内存占用估算](/zh-cn/configuration/memory-estimation/)，详见 [树莓派默认](/zh-cn/configuration/pi5-defaults/) 与 [SSD 调优](/zh-cn/configuration/ssd-tuning/)。

## 存储路径

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `OUTPUT_DIR` | 录制文件保存目录 | `records` |
| `SECRET_DIR` | Cookie、Token、VAPID 密钥目录 | `secrets` |
| `DATABASE_DIR` | bbolt 数据库目录 | `database` |

Docker 部署时建议挂载为卷：

```bash
-v /path/to/records:/app/records
-v /path/to/secrets:/app/secrets
-v /path/to/database:/app/database
```

## 磁盘空间

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `MIN_DISK_SPACE_BYTES` | 录制所需最小磁盘空间（字节） | `5368709120` (5 GB) |

低于此值将拒绝新录制任务。Android 默认使用更低的阈值。

## 订阅房间状态检查

为任一订阅房间开启 **开播通知**（`notify`）或 **自动录制**（`auto_record`）后，后台 `subcheck` 服务会定期检查直播状态。检测到新一场直播时，按房间配置推送通知和/或启动录制。服务启动时先全量检查一次，之后将相关房间分成若干组，在完整周期内分组轮流检查，降低 CPU 峰值。

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `SUBCHECK_CHECK_INTERVAL_SECS` | 完整轮询周期（秒）；各分组在该周期内依次各检查一轮 | `60` |
| `SUBCHECK_SHARD_COUNT` | 检查分组数；每个房间固定归入其中一组，各组在不同时间点检查。默认 60 秒周期、6 组时，大约每 10 秒检查一组 | `6` |

订阅房间较多时可增大分组数，降低单次检查的负载；分组越多，同一房间内两次检查之间的间隔通常也越长（约等于完整周期除以分组数）。使用流程见 [录制直播间](/zh-cn/guides/recording/#订阅房间)。
