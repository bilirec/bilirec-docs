## 服务器绑定

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `HOST` | API 绑定地址，空值监听所有网卡 | (空) |
| `PORT` | API 服务端口 | `8080` |

```bash
export HOST=0.0.0.0    # 监听所有网卡
export PORT=8080
```

## HTTPS

同时设置证书和私钥时，Fiber 默认启用 HTTPS：

```bash
export SERVER_CRT=/path/to/server.crt
export SERVER_KEY=/path/to/server.key
```

<Aside type="caution">
启用 FRP 且 `FRP_HTTPS=true` 时，必须同时设置 `SERVER_CRT` + `SERVER_KEY`，否则协议不匹配导致 FRP 无法工作。详见 [FRP 配置](/zh-cn/configuration/frp/)。
</Aside>

## 反向代理

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `TRUSTED_PROXIES` | 受信任代理 IP/CIDR（逗号分隔） | `161.33.159.26` |
| `PUBLIC_BASE_URL` | 后端公开基址（生成预签名 URL） | (空) |
| `FRONTEND_URL` | 前端 URL（CORS） | `http://localhost:8080` |

`TRUSTED_PROXIES` 默认值 `161.33.159.26` 为公共 FRP 服务 IP。使用 Nginx 等反向代理时，将代理 IP 加入此列表。

`PUBLIC_BASE_URL` 必须包含 `http://` 或 `https://` 前缀，为空时预签名 URL 回退为相对地址。
