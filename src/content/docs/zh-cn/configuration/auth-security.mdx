## REST API 认证

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `USERNAME` | 管理员用户名 | (未设置) |
| `PASSWORD` | 管理员密码 | (未设置) |
| `VIEWER_USERNAME` | 只读访客用户名 | (未设置) |
| `VIEWER_PASSWORD` | 只读访客密码 | (未设置) |
| `JWT_SECRET` | JWT 签名密钥 | `bilirec_secret` |

设置 `USERNAME` 和 `PASSWORD` 后，访问 API 需要先登录。Web 管理页通过登录接口验证账号密码。

`VIEWER_USERNAME` / `VIEWER_PASSWORD` 提供只读账号，写操作仍需 admin 角色。

<Aside type="caution">
暴露到公网（如启用 FRP）后，**建议**设置 `USERNAME` 和 `PASSWORD`。未设置时 API 完全公开访问。
</Aside>

## 运行模式

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `DEBUG` | 调试模式（详见 [开发与调试](/zh-cn/development/debugging/)） | `false` |
| `PRODUCTION_MODE` | 生产模式（远程登录时建议开启） | `false` |
| `SILENT_ACCESS_LOG` | 静默访问日志（仅 4xx/5xx） | `false` |

`DEBUG=true` 时在 `/debug/pprof` 挂载 pprof，需 Authorization 头填入启动日志中的十六进制令牌。详见[开发与调试](/zh-cn/development/debugging/)。

### 何时需要生产模式

| `PRODUCTION_MODE` | 适合场景 |
|-------------------|---------|
| `false`（默认） | 本机 `localhost` / `127.0.0.1` |
| `true` | 通过 [app.bilirec.org](https://app.bilirec.org/) 连接远程 HTTPS 后端（[FRP](/zh-cn/guides/frp/)、自架域名、Nginx 反代等） |

[app.bilirec.org](https://app.bilirec.org/) 和你的后端地址是两个不同网址——**不限于 FRP**，自架 API 也一样。远程管理时建议开启 `PRODUCTION_MODE=true`，浏览器才能正常记住登录状态；否则界面可能反复提示「会话已过期」。

<Aside type="caution">
通过 app.bilirec.org 管理远程后端时，建议 `PRODUCTION_MODE=true` 且 `FRONTEND_URL=https://app.bilirec.org`。与 `*.tunnel.bilirec.org` 同属 `bilirec.org` 根域名，**通常不必**为了登录去允许第三方 Cookie。

若你[自架前端](/zh-cn/development/web/)（如 `https://bilirec.example.com`）并连接**根域名不同**的后端（如 FRP 的 `https://xxx.tunnel.bilirec.org`），除 `PRODUCTION_MODE=true` 与正确的 `FRONTEND_URL` 外，可能还需在浏览器**允许第三方 Cookie**。详见 [远程访问 — 根域名是否相同](/zh-cn/guides/frp/#根域名是否相同)。
</Aside>

**推荐（app.bilirec.org + 远程 HTTPS 后端）：**

```bash
export PRODUCTION_MODE=true
export USERNAME=admin
export PASSWORD=你的强密码
export JWT_SECRET=一个固定的长随机字符串
export FRONTEND_URL=https://app.bilirec.org
```

`JWT_SECRET` 建议在首次部署时设定并长期保持不变；若重启后改了密钥，需要重新登录。

## B 站登录

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `BILIBILI_LOGIN_MODE` | `startup` / `controller` / `anonymous` | `controller` |

详见[快速开始](/zh-cn/guides/quick-start/)中的登录模式说明。

### 直播间信息查询与风控

无论 `BILIBILI_LOGIN_MODE` 为何，**查询直播间是否在播**时程序都会优先使用匿名请求。仅当 B 站 API 返回风控类错误码时，才会自动改用你已登录账号的 Cookie 重试（需 `controller` / `startup` 且已完成扫码登录）。

这意味着：

- **推荐**日常使用 `controller`（默认）：平时轮询不碰账号；只有本地 IP 被 B 站临时限制时才短暂使用 Cookie。
- **`anonymous`**：全程不带 Cookie，被风控时**无法**自动切换，订阅检测与手动查房间信息可能间歇失败。
- **日志**：切换与恢复时后端会输出警告/信息级日志，Web 界面无额外提示。

机制细节见 [录制配置 · 直播间信息查询](/zh-cn/configuration/recording/#直播间信息查询风控自动切换)。
