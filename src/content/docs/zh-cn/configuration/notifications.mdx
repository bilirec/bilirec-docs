## Web Push

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `WEBPUSH_SUBSCRIBER` | VAPID subject（建议 `mailto:you@example.com`） | `mailto:webpush@example.com` |

VAPID 公钥/私钥在启动时自动生成并写入 `SECRET_DIR`（`_webpush_public_key`、`_webpush_private_key`），后续重启复用已有 key。

前端通过 `GET /notify/public-key` 获取公钥后订阅。详见 [通知 API](/zh-cn/api/notify/)。

## SSE 通知

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `NOTIFY_SSE_TOKEN` | SSE 订阅 token | (空字符串) |

为空时 SSE 端点返回 `503`（禁用）。

```http
GET /notify/sse?token=YOUR_NOTIFY_SSE_TOKEN
Accept: text/event-stream
```

推荐用于无法依赖 Google FCM 的场景（如中国大陆网络），Android 后端通过 SSE 获取通知后创建本地通知。建议通过 HTTPS 使用以避免 token 明文暴露。

订阅房间开启 **开播通知** 后，后台会定期检查直播状态并在开播时推送；检查频率与分组方式见 [录制配置](/zh-cn/configuration/recording/#订阅房间状态检查)。
