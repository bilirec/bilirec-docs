## 基本转换

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `CONVERT_TO_MP4` | 录制完成后转为 MP4 | `false` |
| `DELETE_SOURCE_AFTER_CONVERT` | 转换后删除原始源文件 | `false` |
| `NO_CONVERT_IF_INVALID` | 源文件缺少音视频流时是否停止转码；若系统未安装 `ffprobe`，检查会自动放行 | `false` |

启用 `CONVERT_TO_MP4` 后，录制完成的 FLV/TS 文件自动加入转换队列，由后台异步转换为 MP4。

启用 `DELETE_SOURCE_AFTER_CONVERT` 时，源文件删除与转码任务解耦：仅在转换**成功**后异步删除源文件（FFmpeg 与 CloudConvert 共用同一删除器）。删除失败时会指数退避重试，最多 5 次；仍失败则将源文件重命名为在原文件名与扩展名之间插入 `_轉換完成可刪除` 标记，便于手动清理。同一路径不会重复入队删除。

## 本地 FFmpeg

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `FFMPEG_CHECK_INTERVAL_SECS` | 转换任务轮询间隔（秒） | `60` |
| `FFMPEG_MAX_CONCURRENT_TASKS` | 最大并发转换任务数 | `1` |
| `FFMPEG_ALLOW_DURING_RECORDING` | 录制进行中是否允许 FFmpeg 转换 | `false` |
| `FFMPEG_ALLOW_DURING_RECORDING_MAX_ACTIVE_RECORDINGS` | 仅当活跃录制数 `≤` 此值时才允许边录边转；`<1` 表示不设门槛（仍受 `FFMPEG_ALLOW_DURING_RECORDING` 控制） | `1` |

Docker 镜像内置 FFmpeg 8.1.1。桌面环境需在 PATH 中安装 FFmpeg 和 ffprobe。

遗留别名 `FFMPEG_ALLOW_DURING_RECORDING_BELOW_ACTIVE_RECORDINGS` 与 `FFMPEG_ALLOW_DURING_RECORDING_MAX_ACTIVE_RECORDINGS` 等效。

转换失败的任务会进入冷却期，延后 `FFMPEG_CHECK_INTERVAL_SECS × 2` 再重试，避免多任务同时失败后又立刻并发重试。

## CloudConvert 云端

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `CLOUDCONVERT_API_KEY` | API Key（为空则禁用） | (未设置) |
| `CLOUDCONVERT_THRESHOLD` | 使用 CloudConvert 的文件大小阈值（字节） | `1073741824` (1 GB) |
| `CLOUDCONVERT_CHECK_INTERVAL_SECS` | 任务状态轮询间隔（秒） | `180` |
| `CLOUDCONVERT_MAX_CONCURRENT_DOWNLOADS` | 最大并发下载数 | `1` |
| `CLOUDCONVERT_ALLOW_DURING_RECORDING` | 录制进行中是否允许 CloudConvert 转换 | `false` |
| `CLOUDCONVERT_ALLOW_DURING_RECORDING_MAX_ACTIVE_RECORDINGS` | 仅当活跃录制数 `≤` 此值时才允许边录边转；`<1` 表示不设门槛（仍受 `CLOUDCONVERT_ALLOW_DURING_RECORDING` 控制） | `1` |

文件大小 ≥ 阈值时优先使用 CloudConvert。`PUBLIC_BASE_URL` 必须有效，否则 CloudConvert 会拒绝创建任务。
