---
title: 树莓派 5B 默认配置
description: 针对树莓派 5B + microSD 的默认优化参数
---

Bilirec 当前默认配置已针对**树莓派 5B + microSD** 场景优化，在「降低写入次数」和「控制内存占用」之间取平衡。

| 优化项 | 值 | 说明 |
| ----- | ---- | ---- |
| `LIVE_STREAM_WRITER_SYNC_PERIOD_SECS` | `0` | 禁用周期性整文件 fsync；关档时 sync |
| `LIVE_STREAM_WRITER_COLD_CACHE_RELEASE_SECS` | `60` | 录製中周期性释放冷文件页缓存，压低监控曲线 |
| `DROP_FILE_PAGE_CACHE` | `true` | 关档/转码后释放整文件页缓存 |
| `LIVE_STREAM_WRITER_BUFFER_SIZE` | `8 MB` | 降低 flush 频率，优先降低 SD 卡写入频次。@1080p30fps (4.5Mbps) 约每 14.2 秒触发一次满缓冲写入 |
| `LIVE_STREAM_WRITER_FLUSH_PERIOD_SECS` | `10` | 10 秒 flush，优先降低 SD 卡磨损；代价是异常断电时未 flush 数据窗口会略增 |
| `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE` | `64` | 以 512KB chunk 估算，单录制任务约 32MB 队列数据；比 128（约 64MB）更适合 4GB 设备 |
| `SKIP_SMALL_FLUSH` | `true` | 启用小块跳过保护；在达到 `SKIP_SMALL_FLUSH_THRESHOLD`（默认 1MB）之前仅缓存内存，不创建文件，对低比特率流（如 240p）特别有效 |
| `LIVE_STREAM_WRITER_BYTES_POOL_SIZE` | `512 KB` | 与常见 stream chunk 大小一致，减少额外分配与拷贝 |
| `SEQUENTIAL_WRITE` | `true` | 序列化多路写入；多路并发时定时 flush 经 round-robin 错开，有效降低 I/O 峰值，默认启用以保护 microSD |
| `MAX_CONCURRENT_RECORDINGS` | `3` | 默认同时录制上限（保守值）；树莓派 5B + 1 GB 容器、1080p 默认缓冲下，生产环境可验证至 **5 路**（见[性能实测](/zh-cn/guides/performance-benchmark/)），按需调大 |

容器运行时同步：`GOMEMLIMIT=768MiB`、`GOGC=100`。

若录制 4K / 杜比等高码率流，程序会根据 `qn` 自动启用 `*_HIGH` 双池，一般无需改 1080p 基础变量；仅在卡顿时参阅[高码率 / 4K 调参](/zh-cn/configuration/high-bitrate-tuning/)。规划容器内存请参阅[内存占用估算](/zh-cn/configuration/memory-estimation/)。若你的存储介质是 SSD 或 HDD/NAS，请参阅对应的[调优方案](/zh-cn/configuration/ssd-tuning/)。
