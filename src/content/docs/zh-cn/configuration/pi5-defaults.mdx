---
title: 树莓派 5B 默认配置
description: 针对树莓派 5B + microSD 的默认优化参数
---

Bilirec 当前默认配置已针对**树莓派 5B + microSD** 场景优化，在「少写卡、省磨损」和「控制内存占用」之间取平衡。

| 优化项 | 值 | 你会注意到什么 |
| ----- | ---- | -------------- |
| `LIVE_STREAM_WRITER_SYNC_PERIOD_SECS` | `0` | 录制中不反复整文件落盘，只在关档时完整保存；最省 microSD |
| `LIVE_STREAM_WRITER_COLD_CACHE_RELEASE_SECS` | `60` | 长录时 Docker 监控内存不会一路顶满 |
| `DROP_FILE_PAGE_CACHE` | `true` | 关档/转码后释放文件缓存；刚录完立刻播放可能略慢 |
| `LIVE_STREAM_WRITER_BUFFER_SIZE` | `8 MB` | 写盘次数较少；1080p 约每 14 秒写一次 |
| `LIVE_STREAM_WRITER_FLUSH_PERIOD_SECS` | `15` | 每 15 秒把录制缓冲数据推进磁盘，在防止缓冲积聚过多和保持大块写入之间取平衡 |
| `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE` | `64` | 单路约 32 MB 写盘排队；比 128 更省内存 |
| `SKIP_SMALL_FLUSH` | `true` | 极短直播不到 1MB 不建文件，减少空文件 |
| `LIVE_STREAM_WRITER_BYTES_POOL_SIZE` | `512 KB` | 与常见直播块大小一致 |
| `SEQUENTIAL_WRITE` | `true` | 多路同时录时轮流写盘，磁盘峰值更平稳 |
| `MAX_CONCURRENT_RECORDINGS` | `3` | 默认同时录制上限（保守值）；1 GB 容器、1080p 下实测可稳定 **5 路**（见[性能实测](/zh-cn/guides/performance-benchmark/)），按需调大 |

官方 Docker 镜像已内置适合树莓派的运行参数，一般无需再改。

若把 `MAX_CONCURRENT_RECORDINGS` 从默认 `3` 提高到 `5` 路或更多，请同步增大 `LIVE_STREAM_WRITER_FLUSH_PERIOD_SECS`（例如 5 路约 `25` 秒、6 路约 `30` 秒），详见 [I/O 调优 · 多路录制与 flush 周期](/zh-cn/configuration/io-tuning/#多路录制与-flush-周期)。

若录制 4K / 杜比，程序会根据 `qn` 自动启用高画质缓冲，一般无需改 1080p 基础变量；卡顿时参阅 [高码率 / 4K 调参](/zh-cn/configuration/high-bitrate-tuning/)。规划容器内存请参阅 [内存占用估算](/zh-cn/configuration/memory-estimation/)。存储介质为 SSD 或 HDD/NAS 时请参阅 [SSD 调优](/zh-cn/configuration/ssd-tuning/) 或 [HDD/NAS 调优](/zh-cn/configuration/hdd-nas-tuning/)。
