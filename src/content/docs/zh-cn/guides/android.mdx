## 安装

从 [bilirec-mobile Releases](https://github.com/bilirec/bilirec-mobile/releases) 下载并安装 APK。

需要 **Android 10（API 29）** 或更高版本。更低版本系统未经过完整测试，可能无法安装或无法稳定运行前台录制服务。

## 控制中心

App 主界面为 **Bilirec 服务控制中心**，核心元素如下：

| 区域 | 说明 |
|------|------|
| 顶部标题栏 | 显示「Bilirec 服务控制中心」；右上角可切换 **繁 / 简** 界面语言 |
| 中央电源按钮 | 启动或停止 bilirec 系统服务（前台服务 + 内嵌后端） |
| 状态行 | 显示当前服务状态（未启动 / 启动中 / 运行中 / 停止中等） |
| 服务运行后 | 出现 **打开录制管理程序**、**检查系统服务连接** 两个操作按钮 |
| 底部设置入口 | **打开服务启动前设置** 按钮；**仅在服务未运行且未处于启停过程中时可点击** |

## 首次使用与权限

首次打开 App 时，通常会依次遇到以下系统权限与限制：

1. **电池优化豁免** — 弹窗提示将 Bilirec 设为「不受电池限制」。未完成前无法打开设置页，也无法启动服务。点击「前往设置」按系统指引操作后返回 App。
2. **通知权限** — 点击电源按钮启动服务时，若尚未授予通知权限，系统会请求授权。通知用于常驻前台服务；拒绝则无法启动。
3. **（可选）输出路径** — 若要把录制文件保存到 microSD 卡或指定文件夹，请在启动服务前进入设置 → **存储策略** → **更改路径** 选择目录。

服务被系统意外终止时，App 会弹出「Bilirec 系统服务已被终止」提醒，并建议重新启动且确认已关闭省电限制。

## 使用流程

1. （推荐）点击 **打开服务启动前设置**，按需配置输出路径、microSD 卡磨损保护、录制策略等（见下文）
2. 点击中央 **启动** 按钮，等待状态变为「Bilirec 系统服务运行中」
3. 点击 **打开录制管理程序**，跳转到 [app.bilirec.org](https://app.bilirec.org/)（若已安装 PWA 则优先打开 PWA）
4. 在 Web 界面点击「登录」完成 B 站扫码认证，即可订阅房间或开始录制
5. 录制结束后可点击 **停止** 关闭系统服务；也可在通知栏点击「停止服务」

<Tabs>
<TabItem label="检查连接">

若不确定后端是否就绪，可点击 **检查系统服务连接**。App 会请求 `http://127.0.0.1:8080/` 并提示连接正常、响应异常或无响应。

</TabItem>
<TabItem label="前台通知">

服务运行期间，通知栏会常驻显示 CPU / 内存占用。正在录制时，通知正文前会标注「录制中」。若开启了 **激进防休眠模式** 且已生效，通知标题会出现 ⚡（Wi‑Fi 锁定时还带 🛜 或 🌐 图标）。

</TabItem>
</Tabs>

<Aside type="note">
Android 默认连接 `http://127.0.0.1:8080`（等同 `localhost`），且默认不启用账号密码认证。后端在 App 进程内运行，仅本机可访问。
</Aside>

## 服务启动前设置

点击 **打开服务启动前设置** 会弹出可拖曳的底部设置页，自上而下分为以下区块。各选项在下次**启动服务**时注入后端环境变量（输出路径除外，见下表）。

### 一般设置

| App 选项 | 作用 | 对应配置 |
|---------|------|---------|
| **本地通知模式** | 在无法使用 Web Push 时（如部分大陆网络），改由 App 通过 SSE 拉取开播/录制事件并弹出**本地通知** | 启动时自动生成 `NOTIFY_SSE_TOKEN` |
| **激进防休眠模式** | 关屏闲置 1～2 小时后若出现录制断断续续，可尝试开启；录制期间会启用网络锁降低休眠影响 | App 侧开关（非环境变量） |

<Aside type="caution">
开启 **本地通知模式** 后，点击通知**无法**直接跳转到录制管理程序，需手动打开 App 或 PWA。详见 [通知配置](/zh-cn/configuration/notifications/)。
</Aside>

开启 **激进防休眠模式** 会增加耗电，长时间录制建议接电源。通知标题出现 ⚡ 表示防休眠已生效。

### 存储策略

| App 选项 | 作用 | 对应配置 |
|---------|------|---------|
| **输出路径** | 选择录制文件保存目录；未设置时使用 App 私有目录下的默认 `records` | `OUTPUT_DIR`（独立存储，非 managed env） |
| **microSD 卡磨损保护** | 轮流写入录制文件，降低 microSD 同时写入峰值；写入节奏随「同时录制上限」自动调节 | `SEQUENTIAL_WRITE` + `LIVE_STREAM_WRITER_FLUSH_PERIOD_SECS` |

若输出路径选择 **microSD 卡**，请务必开启 **microSD 卡磨损保护**。未开启时后端保持 `SEQUENTIAL_WRITE=false`（并行写入），不会由 androidlib 自动代为开启。

### 录制策略

以下选项通过滑块或分段按钮调整，修改后立即保存，下次启动服务时生效：

| App 选项 | 环境变量 | 可选范围 / 说明 |
|---------|---------|----------------|
| **单次录制时长上限** | `MAX_RECORDING_HOURS` | `0`（不限）～ `12` 小时；滑块显示「不限」即 `0` |
| **启动前最低可用空间** | `MIN_DISK_SPACE_BYTES` | `2` / `5` / `10` GB |
| **下线后服务重试时长** | `MAX_RETRY_MINUTES` | `5`～`30` 分钟（步进 5） |
| **重试成功后的时长计算** | `RECORDING_RECOVERY_DURATION` | **保留**（`preserve`）或 **重置**（`reset`）；断流后若重试成功并恢复录制，可选择沿用开始录制时的总时长，或从重试成功时重新计算录制时长 |
| **同时录制上限** | `MAX_CONCURRENT_RECORDINGS` | `3`～`6` 路；提高会明显增加发热与耗电 |

<Aside type="note">
**启动前最低可用空间** 滑块默认显示 5 GB，但仅在你拖动并松手后才会写入环境变量。若从未调整过，后端仍使用 [Android 默认配置](/zh-cn/configuration/android-defaults/) 中的 `2 GB` 门槛。
</Aside>

### 转换策略

需先开启 **录完自动转为 MP4**，其余转档相关选项才会可用：

| App 选项 | 环境变量 | 说明 |
|---------|---------|------|
| **录完自动转为 MP4** | `CONVERT_TO_MP4` | 使用内嵌 FFmpeg 在录制完成后转 MP4；RAM 少于 8 GB 建议关闭 |
| **转档后删除原始文件** | `DELETE_SOURCE_AFTER_CONVERT` | 转码成功后再异步删除源文件；删除失败会重试，仍失败则重命名为带「转换完成可删除」标记的文件 |
| **录制中允许转换** | `FFMPEG_ALLOW_DURING_RECORDING` | 边录边转，增加读写负载与发热 |
| **允许转档的「最高录制路数」** | `FFMPEG_ALLOW_DURING_RECORDING_MAX_ACTIVE_RECORDINGS` | `0`（不限）～`5` 路；超过时暂停后台转档以优先保证录制 |

转档行为详见 [转换配置](/zh-cn/configuration/conversion/)。

### 开发者选项

面向进阶用户，一般录制无需修改：

| App 选项 | 说明 |
|---------|------|
| **环境参数设置** | 手动添加 `Key` / `Value` 环境变量；**优先级高于**上方各策略开关，覆盖同名字段 |
| **启动日志** | 合并导出 `bootstrap.log` 分片，便于反馈启动或崩溃问题 |

<Aside type="danger">
请勿随意添加不熟悉的环境变量。错误配置可能导致发热、耗电增加、后台被杀或录制不稳定。
</Aside>

完整 App 设置与环境变量对照见上文各区块说明。

## 默认行为

Android 模式下，bilirec 后端会先注入一组移动端默认值，再叠加 App 设置中的环境变量：

- `BILIBILI_LOGIN_MODE=controller` — 通过 Web UI 扫码登录
- `FRP_ENABLED=false` — 手机无需内网穿透
- `SILENT_ACCESS_LOG=true` — 减少日志输出
- 更保守的 I/O / 内存参数（如较小写入缓冲、默认关闭 `SEQUENTIAL_WRITE`）

完整默认值与服务器版差异见 [Android 默认配置](/zh-cn/configuration/android-defaults/)。
