bilirec 内置 FRP 客户端，无需公网 IP 即可在外网安全访问后端管理任务与文件。

## 启用 FRP

最简单的方式——使用官方免费公共服务：

```bash
export FRP_ENABLED=true
```

仅设置 `FRP_ENABLED=true` 即可，程序会自动使用 bilirec.org 官方 FRP 服务。

若要通过 [app.bilirec.org](https://app.bilirec.org/) 登录并管理后端，还需同时开启生产模式并设置账号密码（见下文）。

## 操作流程

1. **启动后端**并确认 FRP 已启用
2. **查看日志**，找到公网访问地址。启动成功后会显示「内网穿透已建立」信息框，**远程地址**一行即公网地址：

   ![启动日志中的公网地址](/images/frp/tunnel-log.png)

3. **打开** [app.bilirec.org](https://app.bilirec.org/)
4. 在登录界面的**服务器地址**栏填入上一步的公网地址，再输入 `USERNAME` 和 `PASSWORD` 登录：

   <img src="/images/frp/login-server-url.png" alt="在登录页填入公网地址" width="400" style="max-width:100%;height:auto;display:block;margin:0 auto" />

<Aside type="caution">
暴露到公网后**务必**设置 `USERNAME` 和 `PASSWORD`，否则 API 将完全公开访问。详见[认证与安全](/zh-cn/configuration/auth-security/)。
</Aside>

## 通过 app.bilirec.org 登录

<Aside type="note">
以下说明**不只适用于 FRP**，也适用于你在 NAS、VPS 等**自架 HTTPS 后端**（例如 `https://api.example.com`、Nginx 反代后的域名）。只要管理页在 [app.bilirec.org](https://app.bilirec.org/)，而后端在**另一个网址**，就可能遇到同样的问题。
</Aside>

管理页和后端地址（无论是 FRP 的 `https://abc123.tunnel.bilirec.org`，还是自架的 `https://api.example.com`）是**两个不同的网站**。登录后，浏览器需要「记住」你已登录；默认配置下往往记不住，于是出现：

- 点了登录，界面显示成功
- 刚进主界面几秒，又弹出「会话已过期，请重新登录」
- 反复登录仍一样

部分浏览器的隐私设置较严，也会拦下登录状态，表现相同。

**推荐配置（通过 app.bilirec.org 连接远程 HTTPS 后端时）：**

```bash
export PRODUCTION_MODE=true
export USERNAME=你的用户名
export PASSWORD=你的密码
export JWT_SECRET=一个固定的长随机字符串
export FRONTEND_URL=https://app.bilirec.org
```

若使用 FRP，再加上 `export FRP_ENABLED=true`。改完后**重启 bilirec**，在登录页填入**当前**的后端地址再登录（FRP 每次重启子域名可能变化）。

**若仍登录不上**，还可以在浏览器里**允许第三方 Cookie**（管理页和后端是不同网站，部分浏览器默认会拦）：

| 浏览器 | 大致路径 |
|--------|---------|
| Chrome / Edge | 设置 → 隐私和安全 → 第三方 Cookie → 选择「允许第三方 Cookie」，或将 `app.bilirec.org` 与你的**后端域名**（如 `tunnel.bilirec.org`、`api.example.com`）加入允许名单 |
| Firefox | 设置 → 隐私与安全 → 在「增强型跟踪保护」中为 `app.bilirec.org` 关闭保护，或放宽 Cookie 限制 |
| Safari（iOS / macOS） | 设置 → Safari → 关闭「阻止跨站跟踪」，或在访问 `app.bilirec.org` 时允许该网站使用 Cookie |

自架后端时，把域名换成你实际使用的地址。改完浏览器设置后，重新打开管理页再登录一次。

本机开发（`http://localhost:8080`）不受此影响。详见 [认证与安全 — 生产模式](/zh-cn/configuration/auth-security/#何时需要生产模式) 与 [常见问题](/zh-cn/guides/faq/#用-appbilirecorg-连远程后端一直提示会话过期)。

## 自建 FRP 服务器

若你有自己的 FRP 服务，可自定义配置：

```bash
export FRP_ENABLED=true
export FRP_SERVER=your-frp-ip:7000
export FRP_TOKEN=your-token
export FRP_BASE_DOMAIN=your-domain.com
```

启动日志会显示 `FRP enabled in custom-selfhost mode`。

技术细节、协议约束与完整参数说明见 [FRP 配置](/zh-cn/configuration/frp/)。
