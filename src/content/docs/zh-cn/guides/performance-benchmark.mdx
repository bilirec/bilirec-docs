- 设备：树莓派 5B 16GB
- 场景：默认配置，同时录制多个直播间
- 指标：CPU 峰值占用、内存占用

## 数据汇总

| 并发路数 | CPU 峰值占用 | 内存占用 |
|---------|------------|---------|
| 初始闲置 | ~0.0% | ~7MB |
| 1 路并发 | ~0.8% | ~42MB |
| 2 路并发 | ~2.6% | ~76MB |
| 3 路并发 | ~3.0% | ~96MB |
| 4 路并发 | ~3.4% | ~129MB |
| 5 路并发 | ~4.1% | ~142MB |
| 恢复闲置 | ~0.0% | ~54MB |

<Aside type="note">
bilirec 使用了较多内存池以降低 GC 压力，因此部分内存会被池化复用而非立即归还系统。默认配置也偏向减少 microSD 写入磨损，内存占用会更高一些。`SKIP_SMALL_FLUSH=true` 时，是否落盘由 `SKIP_SMALL_FLUSH_THRESHOLD` 决定（默认 1MB，独立于 `LIVE_STREAM_WRITER_BUFFER_SIZE`）。若要进一步压低占用，请参考[配置与调优](/zh-cn/configuration/overview/)。
</Aside>
