本文记录 bilirec 在低配设备上的实测表现，便于你评估是否适合自己的部署环境。

## 测试环境

- 设备：树莓派 5B 16GB
- 容器：Docker（1 GB 内存限制）
- 场景：默认配置，每路直播 1080p，同时录制多个直播间
- 指标：CPU 峰值占用、内存峰值占用

<Aside type="note">
以下数据仅供参考（实测快照，改配置或版本后会有偏差）。下表为默认配置下的**常见实测峰值**；按环境变量规划容器上限请用 [内存占用估算](/zh-cn/configuration/memory-estimation/)（偏保守，通常高于本表）。
</Aside>

## 数据汇总

| 并发路数 | CPU 峰值占用 | 内存峰值占用 |
|---------|------------|-------------|
| 初始闲置 | ~0.0% | ~7MB |
| 1 路并发 | ~1.8% | ~72MB |
| 2 路并发 | ~2.7% | ~117MB |
| 3 路并发 | ~3.6% | ~141MB |
| 4 路并发 | ~4.1% | ~165MB |
| 5 路并发 | ~4.9% | ~192MB |
| 恢复闲置 | ~0.0% | ~54MB |

<Aside type="note">
bilirec 使用了大量内存池以降低 GC 压力，停止录制后部分内存会保留复用（见上表「恢复闲置」），以减轻 GC 压力。内存占用与缓冲配置、录制路数相关；**真实占用请按 [内存占用估算](/zh-cn/configuration/memory-estimation/) 计算**，勿直接套用上表数字。
</Aside>

<Aside type="tip">
**常见问题：** 如何用实测数据评估自己的设备、停录后内存为何仍偏高——见 [常见问题](/zh-cn/guides/faq/#性能与容量规划)。
</Aside>
