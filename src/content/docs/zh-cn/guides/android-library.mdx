若你正在将 `libbilirec.so` 集成到自己的 Android 项目，该库导出两个 C 方法：

- `Start(configJson)` — 启动服务（传入 JSON 配置），返回 `C.int`
- `Stop()` — 停止服务，返回 `C.int`

## Start 配置格式

```json
{
  "basePath": "/data/user/0/your.app/files",
  "env": {
    "PORT": "8080",
    "HOST": "127.0.0.1",
    "FRONTEND_URL": "https://app.bilirec.org",
    "USERNAME": "admin",
    "PASSWORD": "changeme",
    "MAX_CONCURRENT_RECORDINGS": "3",
    "SEQUENTIAL_WRITE": "false"
  }
}
```

- `basePath` **必填** — 数据目录根路径
- `env` **可选** — 覆盖默认环境变量（含 `SEQUENTIAL_WRITE`、`OUTPUT_DIR`、`CONVERT_TO_MP4` 等）

## 移动端默认行为

Android 模式下会先注入一组移动端默认值，再套用 `env` 覆盖。默认行为如下（可覆盖）：

- `BILIBILI_LOGIN_MODE=controller`
- `FRP_ENABLED=false`
- `SILENT_ACCESS_LOG=true`

完整参数见 [Android 默认配置](/zh-cn/configuration/android-defaults/)。

## 构建 libbilirec.so

```bash
# Linux 主机
make android os=linux

# Windows 主机
make android os=windows
```

需要 Android NDK 24+ 及 `./lib/<abi>/libffmpegkit.so`。输出目录：`dist/android/<abi>/libbilirec.so`。

详见[从源码构建](/zh-cn/development/build-from-source/)。

## FFmpeg 运行时

`libbilirec.so` 依赖 FFmpeg 进行 MP4 转档。bilirec-mobile 通过 Gradle 依赖 `com.antonkarpenko:ffmpeg-kit-min:2.1.0` 提供运行时。自定义集成时须自行提供兼容的 FFmpeg 库。
