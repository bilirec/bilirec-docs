[bilirec-web](https://github.com/bilirec/bilirec-web) 是 bilirec 的通用 Web 管理介面，線上地址為 [app.bilirec.org](https://app.bilirec.org/)。它是一個獨立的前端項目，通過 REST API 連接任意 bilirec 後端實例。

面向最終用戶的使用說明見 [Web 介面（PWA）](/zh-tw/guides/web-ui/)。API 端點詳見 [開發者集成](/zh-tw/api/overview/)。

## 項目定位

- **主要目標**：為 bilirec 後端提供移動優先的 PWA 管理介面。
- **技術方案**：React SPA + Service Worker（PWA）+ Axios 調用後端 REST API。
- **部署方式**：靜態資源託管（官方實例部署於 Cloudflare Workers）；後端須單獨運行。
- **多後端支持**：登入頁可設定伺服器地址，地址保存在 `localStorage`，並同步到 IndexedDB 供 Service Worker 使用（鍵名均為 `server-url`）。

<Aside type="note">
線上 `app.bilirec.org` 與自架 **HTTPS** 前端受同一套瀏覽器混合內容策略：只能連接本機 HTTP 後端（`localhost` / `127.0.0.1`）；遠端 HTTP 局域網地址會被攔截，須使用 HTTPS 隧道（如 [FRP](/zh-tw/guides/frp/)），或把前後端放在同一 HTTPS 反代後。詳見 [Web 介面](/zh-tw/guides/web-ui/#連接自架後端)。
</Aside>

## 架構概覽

```text
┌─────────────────────────────────────────────────────────┐
│  React 應用（src/App.tsx）                                 │

│  LeftSidebar（桌面）/ BottomNav（移動）                    │
└───────────────────────┬─────────────────────────────────┘
                        │ Axios（apiClient）
┌───────────────────────▼─────────────────────────────────┐
│  bilirec 後端 REST API                                     │
│  預設 http://localhost:8080                                │
└─────────────────────────────────────────────────────────┘

│  Service Worker（src/sw.ts）                               │
```

## 本地開發

### 前置條件

- Node.js 與 **pnpm**
- 一個正在運行的 bilirec 後端（本地或遠端 HTTPS）

### 啟動項目

```bash
pnpm install
pnpm dev
```

開發伺服器預設監聽 `http://localhost:3000`（`vite.config.ts` 中 `server.port = 3000`，`host: true` 允許局域網存取開發伺服器本身）。

在登入頁填入後端地址。本地開發時若後端跑在本機，使用 `http://localhost:8080` 或 `http://127.0.0.1:8080`。

### 建置與預覽

```bash
pnpm build    # tsc + vite build，輸出 dist/
pnpm preview  # 本地預覽生產建置
```

## 自行部署（Cloudflare）

官方 [app.bilirec.org](https://app.bilirec.org/) 通過 Cloudflare Workers 託管 `dist/` 靜態資源（見 `wrangler.jsonc`）。你也可以 fork 倉庫後一鍵部署到自己的 Cloudflare 帳號：

[![Deploy to Cloudflare](https://deploy.workers.cloudflare.com/button)](https://deploy.workers.cloudflare.com/?url=https://github.com/bilirec/bilirec-web)

<Aside type="caution" title="部署注意事項">

| 事項 | 說明 |
|------|------|
| 後端 `FRONTEND_URL` | 部署完成後，將 bilirec 後端的 `FRONTEND_URL` 設為你的前端 HTTPS 域名（CORS 與 Cookie 認證）。見 [伺服器與網路設定](/zh-tw/configuration/server-network/) |
| 自架前端 + 不同根域名後端 | 若前端域名與後端（如 FRP 的 `*.tunnel.bilirec.org`）**根域名不同**，除 `PRODUCTION_MODE=true` 外，可能還需在瀏覽器允許第三方 Cookie。官方 `app.bilirec.org` 連 FRP 一般無此問題。見 [FRP — 根域名是否相同](/zh-tw/guides/frp/#根域名是否相同) |
| 須使用 HTTPS | PWA 安裝與 Web Push 需要 HTTPS。任何 HTTPS 前端（含自架）都受瀏覽器混合內容限制：只能直連本機 HTTP（`localhost` / `127.0.0.1`），不能直連局域網 HTTP 地址；遠端後端須為 HTTPS（如 [FRP](/zh-tw/guides/frp/)）或把前後端放在同一 HTTPS 反代後 |
| 與官方前端取捨 | `app.bilirec.org` 適合本機、Android 內嵌後端或 FRP HTTPS 隧道；自架前端適合自定義域名，或前後端同站 HTTPS 反代等場景 |
| Fork 行為 | 按鈕會在你的 GitHub 創建倉庫 fork 並綁定 Workers Builds，後續 push 自動部署 |
| 後端須單獨運行 | 前端僅為靜態資源，bilirec 錄製後端須另行[安裝與啟動](/zh-tw/guides/installation/) |
| 首次連接 | 打開自架前端 → 填入 bilirec 後端地址 → 必要時登入 → 完成 B 站掃碼。見 [快速開始](/zh-tw/guides/quick-start/) |

</Aside>

### 手動部署

自行部署時也可不使用 Cloudflare 按鈕：

1. 執行 `pnpm build` 生成 `dist/`。
2. 將 `dist/` 部署到任意靜態託管（Cloudflare Pages、Nginx、對象儲存 + CDN 等），或使用 `pnpm deploy`（`wrangler deploy`）。
3. **須使用 HTTPS**，否則 PWA 安裝與 Web Push 等功能受限。
4. 確保後端 `FRONTEND_URL` 包含你的前端域名（CORS）。見 [伺服器與網路設定](/zh-tw/configuration/server-network/)。

## 認證與後端連接

- 登入時 `apiClient.setBaseURL(url)` 設定後端地址，並寫入 `localStorage` 的 `server-url`；同時寫入 IndexedDB（`shared-store`），供 Service Worker 讀取。
- 若提供 `USERNAME` / `PASSWORD`，調用 `POST /login`，JWT 通過 Cookie 返回（不在 JavaScript 中儲存令牌）。僅當後端 `PRODUCTION_MODE=true` 時 Cookie 為 HttpOnly + Secure；本機開發預設可不設生產模式。
- 未設定帳號密碼時，前端會嘗試無認證存取；後端返回 401 則提示需要登入。

## PWA 與 Service Worker

- 建置策略：`vite-plugin-pwa` 的 **injectManifest**，入口為 `src/sw.ts`。
- `registerType: 'prompt'` — 由瀏覽器決定是否顯示安裝提示。
- Service Worker 處理 Web Push 事件，並從 IndexedDB 讀取 `server-url` 以定位推送訂閱端點。
- 用戶安裝說明見 [Web 介面 — 安裝 PWA](/zh-tw/guides/web-ui/#安裝-pwa)。

<Aside type="tip">
修改 Service Worker 後須重新 `pnpm build` 並確保用戶刷新或更新 PWA，新 SW 才會生效。
</Aside>

## 實時刷新

錄製、訂閱與轉碼列表約每 **5 秒**自動刷新；頁面不可見（標籤頁隱藏）時暫停這些刷新以節省資源。磁碟佔用另有獨立刷新節奏；檔案列表按需刷新。
