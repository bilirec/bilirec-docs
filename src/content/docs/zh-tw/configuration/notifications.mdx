## Web Push

| 變數 | 說明 | 預設值 |
|------|------|--------|
| `WEBPUSH_SUBSCRIBER` | VAPID subject（建議 `mailto:you@example.com`） | `mailto:webpush@example.com` |

VAPID 公鑰/私鑰在啟動時自動生成並寫入 `SECRET_DIR`（`_webpush_public_key`、`_webpush_private_key`），後續重啟複用已有 key。

前端通過 `GET /notify/public-key` 獲取公鑰後訂閱。詳見 [通知 API](/zh-tw/api/notify/)。

## SSE 通知

| 變數 | 說明 | 預設值 |
|------|------|--------|
| `NOTIFY_SSE_TOKEN` | SSE 訂閱 token | (空字符串) |

為空時 SSE 端點返回 `503`（禁用）。

```http
GET /notify/sse?token=YOUR_NOTIFY_SSE_TOKEN
Accept: text/event-stream
```

推薦用於無法依賴 Google FCM 的場景（如中國大陸網路）。bilirec-mobile 開啟「本地通知模式」後，App 通過本機 SSE 接收事件並彈出本地通知（走 `127.0.0.1`，無需 HTTPS）。若把 SSE 暴露到遠端網路，建議使用 HTTPS，避免 token 明文傳輸。

訂閱房間開啟 **開播通知** 後，後台會定期檢查直播狀態並在開播時推送；檢查頻率與分組方式見 [錄製設定](/zh-tw/configuration/recording/#訂閱房間狀態檢查)。
