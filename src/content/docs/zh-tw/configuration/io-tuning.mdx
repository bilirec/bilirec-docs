---
title: I/O 調優
description: 緩衝區、flush、sync 與寫入序列化參數
---

這些參數控制 bilirec 的磁盤 I/O 行為，對 microSD 卡壽命和多路併發性能有顯著影響。

## 通用緩衝區

| 變量 | 說明 | 默認值 |
|------|------|--------|
| `UPLOAD_BUFFER_SIZE` | 上傳/外部傳輸緩衝區（字節） | `5242880` (5 MB) |
| `DOWNLOAD_BUFFER_SIZE` | 檔案下載緩衝區（字節） | `5242880` (5 MB) |
| `STREAM_WRITER_BUFFER_SIZE` | 流寫入器緩衝區（字節） | `1048576` (1 MB) |

## 直播流讀取

這組參數用於**直播流讀取階段**（網絡讀取與讀取管道緩衝）。bilirec 為 **1080p** 與 **高畫質（2K/4K/杜比）** 各準備了一套記憶體池，錄製時根據實際流的 `qn` 自動選用。

| 變量 | 說明 | 默認值 |
|------|------|--------|
| `READ_STREAM_BYTES_POOL_SIZE` | 1080p 及以下 FLV 每次讀取的緩衝塊大小（字節） | `524288` (512 KB) |
| `READ_STREAM_BYTES_POOL_SIZE_HIGH` | 2K/4K FLV 讀塊大小（`qn ≥ 20000` 時自動啟用） | `1048576` (1 MB) |
| `READ_STREAM_CHAN_BUFFER_SIZE` | 1080p 及以下讀側 channel 深度（chunk 數） | `16` |
| `READ_STREAM_CHAN_BUFFER_SIZE_HIGH` | 2K/4K 讀側 channel 深度（`qn ≥ 20000` 時自動啟用） | `48` |

補充說明：

- **懶加載**：只有真的在錄高畫質時才初始化 `*_HIGH` 池；不錄 4K 時不會常駐佔記憶體
- **HLS 讀側**：不走 bytes pool，但 `qn ≥ 20000` 時同樣使用 `READ_STREAM_CHAN_BUFFER_SIZE_HIGH`
- **保溫上限**：讀側 bytes pool 空閒複用塊約為 `READ_STREAM_CHAN_BUFFER_SIZE`（或 `_HIGH`）的 1/4；在途記憶體估算公式不變，詳見 [記憶體佔用估算](/zh-tw/configuration/memory-estimation/)

## 直播流寫入檔案

這組參數用於**將直播流寫入檔案**階段（flush/sync/隊列）。

| 變量 | 說明 | 默認值 |
|------|------|--------|
| `LIVE_STREAM_WRITER_BUFFER_SIZE` | 實時流寫入緩衝區（字節） | `8388608` (8 MB) |
| `LIVE_STREAM_WRITER_SYNC_PERIOD_SECS` | 週期性 fsync 週期（秒），`0`=禁用 | `0` |
| `LIVE_STREAM_WRITER_FLUSH_PERIOD_SECS` | 週期性 flush 週期（秒） | `10` |
| `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE` | 通道緩衝區（數據塊數）；1080p 與 4K 共用 | `64` |
| `LIVE_STREAM_WRITER_BYTES_POOL_SIZE` | 1080p 及以下寫側緩衝塊大小（字節） | `524288` (512 KB) |
| `LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH` | 2K/4K 寫側塊大小（`qn ≥ 20000` 時自動啟用） | `1048576` (1 MB) |

### 關鍵參數說明

- **`LIVE_STREAM_WRITER_SYNC_PERIOD_SECS=0`** — 禁用週期性 fsync（最昂貴的操作），僅在錄製結束時 sync。大幅降低 SD 卡磨損，但意外斷電可能丟失最後一段數據
- **`LIVE_STREAM_WRITER_BUFFER_SIZE=8MB`** — 更大緩衝降低 flush 頻率。@1080p30fps (4.5Mbps) 約每 14 秒觸發一次滿緩衝寫入
- **`LIVE_STREAM_WRITER_FLUSH_PERIOD_SECS=10`** — 10 秒 flush 一次，優先降低 SD 卡磨損
- **寫側保溫上限**：`LIVE_STREAM_WRITER_BYTES_POOL_SIZE` 各檔位的空閒複用塊約為 `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE` 的 1/8

## 2K/4K 雙池一覽

| 畫質 | 判定（`qn`） | FLV 讀塊 | 讀側 channel | 寫側塊 |
| ---- | ------------ | -------- | ------------ | ------ |
| 1080p 及以下 | `< 20000` | `READ_STREAM_BYTES_POOL_SIZE`（512 KB） | `READ_STREAM_CHAN_BUFFER_SIZE`（16） | `LIVE_STREAM_WRITER_BYTES_POOL_SIZE`（512 KB） |
| 2K / 4K / 杜比 | `≥ 20000` | `READ_STREAM_BYTES_POOL_SIZE_HIGH`（1 MB） | `READ_STREAM_CHAN_BUFFER_SIZE_HIGH`（48） | `LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH`（1 MB） |

高規格專用變量共三個：`READ_STREAM_BYTES_POOL_SIZE_HIGH`、`READ_STREAM_CHAN_BUFFER_SIZE_HIGH`、`LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH`（讀寫塊大小建議保持一致）。詳見[高碼率 / 4K 調參](/zh-tw/configuration/high-bitrate-tuning/)。

## 寫入保護

| 變量 | 說明 | 默認值 |
|------|------|--------|
| `SKIP_SMALL_FLUSH_THRESHOLD` | 啟用 `SKIP_SMALL_FLUSH` 後，延遲創建檔案並緩存記憶體的閾值（字節） | `1048576` (1 MB) |
| `SKIP_SMALL_FLUSH` | 啟用後在達到閾值前僅緩存記憶體，不創建檔案 | `true` |
| `SEQUENTIAL_WRITE` | 全局 flush 鎖序列化多路寫入 | `true` |

- **`SKIP_SMALL_FLUSH`** — 在達到 `SKIP_SMALL_FLUSH_THRESHOLD`（默認 1MB）之前僅緩存記憶體、不創建檔案；對低比特率流（如 240p）特別有效。該閾值獨立於 `LIVE_STREAM_WRITER_BUFFER_SIZE`
- **`SEQUENTIAL_WRITE`** — 多路併發寫入同一物理磁盤時顯著降低 I/O 峰值；啟用後多路錄製的定時 flush 會經 round-robin 錯開。SSD 環境建議禁用
