遠端存取是指：讓運行在內網（如家庭寬帶、NAS、樹莓派）的服務在沒有公網 IP 的情況下，也能被外網存取。
對 bilirec 來說，這通常通過 FRP 內網穿透實現；啟用後，你可以在外出時通過公網地址存取自己的錄製後端。

啟用 FRP (`FRP_ENABLED=true`) 時，bilirec 作為遠端存取客戶端連接到 FRP 伺服器。

僅設定 `FRP_ENABLED=true` 時，程式預設使用官方免費公共 FRP 服務；設定 `FRP_SERVER`、`FRP_BASE_DOMAIN` 或 `FRP_TOKEN` 時切換為自定義設定。

## 基礎參數

| 變數 | 說明 | 預設值 |
|------|------|--------|
| `FRP_ENABLED` | 是否啟用 FRP | `false` |
| `FRP_SERVER` | FRP 伺服器地址（`host:port`） | `tunnel.bilirec.org:7000` |
| `FRP_TOKEN` | FRP 認證 Token | (空) |
| `FRP_BASE_DOMAIN` | 公網基礎域名 | `tunnel.bilirec.org` |
| `TRUSTED_PROXIES` | 受信任代理列表 | `161.33.159.26` |

## 協議設定

| 變數 | 說明 | 預設值 |
|------|------|--------|
| `FRP_HTTPS` | FRP 代理協議（`true`=HTTPS） | `false` |
| `FRP_SCHEME_HTTPS` | 公網 URL scheme | `true` |

<Aside type="caution">
**協議約束：**
- `FRP_ENABLED=true` 且 `FRP_HTTPS=true` 時，**必須**同時設定 `SERVER_CRT` + `SERVER_KEY`
- 同時設定 `SERVER_CRT` + `SERVER_KEY` 且 `FRP_ENABLED=true` 時，`FRP_HTTPS` **不能**為 `false`

違反約束會導致協議不匹配，程式在啟動階段直接報錯退出。
</Aside>

## 設定示例

**官方公共服務（推薦）：**

```bash
export FRP_ENABLED=true
```

通過 [app.bilirec.org](https://app.bilirec.org/) 遠端登入時（**不限於 FRP**，自架 HTTPS API 也一樣），還需 `PRODUCTION_MODE=true` 及帳號密碼，否則可能反覆提示會話過期。見 [認證與安全 — 生產模式](/zh-tw/configuration/auth-security/#何時需要生產模式)。

**app.bilirec.org + 官方 FRP（完整示例）：**

```bash
export FRP_ENABLED=true
export PRODUCTION_MODE=true
export USERNAME=admin
export PASSWORD=你的強密碼
export JWT_SECRET=一個固定的長隨機字符串
```

**自定義 FRP：**

```bash
export FRP_ENABLED=true
export FRP_SERVER=your-frp-ip:7000
export FRP_TOKEN=your-token
export FRP_BASE_DOMAIN=your-domain.com
export FRP_HTTPS=false
export FRP_SCHEME_HTTPS=true
```

## 啟動日誌

```
FRP enabled in official-public mode    # 官方公共服務
FRP enabled in custom-selfhost mode    # 自定義 FRP
```

操作流程見 [遠端存取指南](/zh-tw/guides/frp/)。

<Aside type="tip">
**常見問題：** 內網穿透與 HTTPS 開關要一致；地址請寫成 `主機:端口`；自架 FRP 需填 Token；通過 app.bilirec.org 登入需 `PRODUCTION_MODE=true`。詳見 [常見問題](/zh-tw/guides/faq/)。
</Aside>
