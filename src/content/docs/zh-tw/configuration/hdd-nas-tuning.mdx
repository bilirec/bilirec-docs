機械硬碟最怕**磁頭來回找位置**。多路同時寫很多小檔案時，NAS 容易卡頓。調參核心是：**一次寫大一點**、**多路輪流寫**。

## 推薦設定

```bash
export LIVE_STREAM_WRITER_SYNC_PERIOD_SECS=45
export LIVE_STREAM_WRITER_COLD_CACHE_RELEASE_SECS=0
export DROP_FILE_PAGE_CACHE=false
export SKIP_SMALL_FLUSH=false
export SEQUENTIAL_WRITE=true
export LIVE_STREAM_WRITER_BUFFER_SIZE=16777216
export MAX_CONCURRENT_RECORDINGS=5
```

| 參數 | 調整 | 你會注意到什麼 |
|------|------|----------------|
| `LIVE_STREAM_WRITER_SYNC_PERIOD_SECS=45` | 適當拉長整檔案落盤間隔 | 減少打斷順序寫入；`COLD=0` 時 整檔案落盤 後不會釋放頁緩存 |
| `LIVE_STREAM_WRITER_COLD_CACHE_RELEASE_SECS=0` | 關閉錄製中清理檔案緩存 | NAS 記憶體通常更大，回放更順 |
| `DROP_FILE_PAGE_CACHE=false` | 關檔時不釋放檔案緩存 | 錄完立刻播放更快 |
| `SKIP_SMALL_FLUSH=false` | 不跳過極短直播保護 | 更早上盤 |
| `SEQUENTIAL_WRITE=true` | 多路輪流寫盤 | 磁頭尋道更少，多路更穩 |
| `LIVE_STREAM_WRITER_BUFFER_SIZE=16MB` | 單次寫入更大 | 檔案在盤上更連續 |
| `MAX_CONCURRENT_RECORDINGS=5` | 適度並發 | 視硬盤轉速與記憶體調整 |

<Aside type="tip">
**混合部署建議：**
- **ZFS 檔案系統：** 可將 `LIVE_STREAM_WRITER_SYNC_PERIOD_SECS` 設為 `0` 或 `60`，少做應用層反覆落盤
- **與私有云服務共存：** 若硬盤同時跑 Cloudreve、影音庫等，建議保持 `SEQUENTIAL_WRITE=true`，避免多服務搶磁頭
</Aside>
