機械硬盤的物理弱點是**磁頭尋道延遲**。多路併發錄製時頻繁隨機寫入會導致嚴重卡頓。調整核心在於**合併大塊數據**並**強制順序寫入**。

## 推薦設定

```bash
export LIVE_STREAM_WRITER_SYNC_PERIOD_SECS=45
export SKIP_SMALL_FLUSH=false
export SEQUENTIAL_WRITE=true
export LIVE_STREAM_WRITER_BUFFER_SIZE=16777216
export MAX_CONCURRENT_RECORDINGS=5
```

| 參數 | 調整 | 原因 |
|------|------|------|
| `SYNC_PERIOD_SECS=45` | 適當拉長 sync | 避免頻繁打斷順序寫入 |
| `SKIP_SMALL_FLUSH=false` | 保障完整性 | — |
| `SEQUENTIAL_WRITE=true` | **保持全局寫鎖** | 多路併發強制順序寫入，降低尋道延遲 |
| `BUFFER_SIZE=16MB` | 增大單次落盤 | 檔案在磁盤上分佈更連續 |
| `MAX_CONCURRENT_RECORDINGS=5` | 適度併發 | 視硬盤轉速調整 |

<Aside type="tip">
**混合部署建議：**
- **ZFS 檔案系統：** 可將 `SYNC_PERIOD_SECS` 設為 `0` 或 `60`，交由 ARC 緩存接管
- **與私有云服務共存：** 若硬盤同時運行 Cloudreve、影音服務器等，強烈建議保持 `SEQUENTIAL_WRITE=true`，避免磁頭爭搶
</Aside>
