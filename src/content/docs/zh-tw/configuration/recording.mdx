## 併發與時長

| 變量 | 說明 | 默認值 |
|------|------|--------|
| `MAX_CONCURRENT_RECORDINGS` | 最大同時錄製數 | `3` |
| `MAX_RECORDING_HOURS` | 單次錄製最長時間（小時） | `5` |
| `MAX_RECOVERY_ATTEMPTS` | 單次錄製最大重連嘗試次數 | `5` |
| `MAX_RETRY_MINUTES` | 直播中斷後判斷容忍時間（分鐘） | `10` |

`MAX_CONCURRENT_RECORDINGS` 直接影響內存峰值。樹莓派默認 `3`（1080p 默認緩衝下可驗證至 5 路），SSD 環境可放寬至 `10`。規劃容器上限請參閱 [內存佔用估算](/zh-tw/configuration/memory-estimation/)，詳見 [樹莓派默認](/zh-tw/configuration/pi5-defaults/) 與 [SSD 調優](/zh-tw/configuration/ssd-tuning/)。

## 存儲路徑

| 變量 | 說明 | 默認值 |
|------|------|--------|
| `OUTPUT_DIR` | 錄製檔案保存目錄 | `records` |
| `SECRET_DIR` | Cookie、Token、VAPID 密鑰目錄 | `secrets` |
| `DATABASE_DIR` | bbolt 數據庫目錄 | `database` |

Docker 部署時建議掛載為卷：

```bash
-v /path/to/records:/app/records
-v /path/to/secrets:/app/secrets
-v /path/to/database:/app/database
```

## 磁盤空間

| 變量 | 說明 | 默認值 |
|------|------|--------|
| `MIN_DISK_SPACE_BYTES` | 錄製所需最小磁盤空間（字節） | `5368709120` (5 GB) |

低於此值將拒絕新錄製任務。Android 默認使用更低的閾值。
