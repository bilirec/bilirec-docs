默認的 512 KB（1080p）+ 三個 `*_HIGH` 變量（高畫質）已覆蓋大多數場景：**錄 1080p 不用改**；錄 4K / 杜比時傳對應 `qn` 即可，程序會自動啟用高規格設定。

只有出現卡頓、記憶體不足、或單塊數據（FLV 讀塊 / HLS segment）明顯偏大時，再按下表微調。

## 2K/4K 雙池機制

bilirec 為 **1080p** 與 **高畫質（2K/4K/杜比）** 各準備了一套記憶體池。錄製時根據實際流的 `qn` 自動選用，無需在設定裡手動切換「4K 模式」。

| 畫質 | 判定（`qn`） | FLV 讀塊 | 讀側 channel | 寫側塊 |
| ---- | ------------ | -------- | ------------ | ------ |
| 1080p 及以下 | `< 20000` | `READ_STREAM_BYTES_POOL_SIZE`（512 KB） | `READ_STREAM_CHAN_BUFFER_SIZE`（16） | `LIVE_STREAM_WRITER_BYTES_POOL_SIZE`（512 KB） |
| 2K / 4K / 杜比 | `≥ 20000` | `READ_STREAM_BYTES_POOL_SIZE_HIGH`（1 MB） | `READ_STREAM_CHAN_BUFFER_SIZE_HIGH`（48） | `LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH`（1 MB） |

高規格專用變量共三個：`READ_STREAM_BYTES_POOL_SIZE_HIGH`、`READ_STREAM_CHAN_BUFFER_SIZE_HIGH`、`LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH`（讀寫塊大小建議保持一致）。

| 要點 | 說明 |
| ---- | ---- |
| 懶加載 | 只有真的在錄高畫質時才初始化 `*_HIGH` 池；不錄 4K 時不會常駐佔記憶體 |
| HLS 讀側 | 不走 bytes pool，但 `qn ≥ 20000` 時同樣使用 `READ_STREAM_CHAN_BUFFER_SIZE_HIGH` |
| 寫側 channel | `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE` 不分檔，1080p 與 4K 共用 |

## 常見場景

| 場景 | 要不要改設定 | 建議 |
| ---- | ------------ | ---- |
| 1080p 原畫 | 否 | 保持默認 |
| 首次錄 4K / 杜比 | 通常否 | API 傳 `qn=20000`（或 `30000`），自動走高畫質池 |
| 4K FLV 塊被拆太碎、分配頻繁 | 是 | 同時加大 `READ_STREAM_BYTES_POOL_SIZE_HIGH` 與 `LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH`（兩者保持一致） |
| 4K HLS segment 很大 | 是 | 主要加大 `LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH` |
| 4K 網絡抖動、讀側阻塞 | 是 | 加大 `READ_STREAM_CHAN_BUFFER_SIZE_HIGH`（默認 48） |
| 4GB 設備多路 4K | 視情況 | 優先減小 `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE`（如 `48`）或 `MAX_CONCURRENT_RECORDINGS` |

## 推薦起點

三個 `*_HIGH` 默認值通常已夠用，下表供參考：

| 場景 | `READ_STREAM_BYTES_POOL_SIZE_HIGH` | `READ_STREAM_CHAN_BUFFER_SIZE_HIGH` | `LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH` | `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE` |
| ---- | ---------------------------------- | ----------------------------------- | ----------------------------------------- | ------------------------------------- |
| 4K / 杜比 FLV 單路 | `1048576` (1 MB) | `48` | `1048576` (1 MB) | `64` |
| 4K / 杜比 HLS 單路 | 對 FLV 讀塊無影響 | `48` | `1048576` (1 MB) | `64` |
| SSD / NAS 單路極高碼率 | `1048576`～`2097152` | `48`～`64` | 與讀塊 `*_HIGH` 相同 | `64`～`128` |

<Aside type="caution">
調 4K 請改帶 `_HIGH` 後綴的變量。不要動不帶 `_HIGH` 的基礎變量——那是 1080p 用的，改大隻會讓所有錄製都多佔記憶體。
</Aside>

## 示例（4K 默認仍不夠用時）

```bash
export READ_STREAM_BYTES_POOL_SIZE_HIGH=2097152       # 2 MB，與寫側保持一致
export READ_STREAM_CHAN_BUFFER_SIZE_HIGH=48
export LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH=2097152
export LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE=64       # 多路 4K 可降到 48
```

**記憶體粗算（單路上限）：** 見 [記憶體佔用估算](/zh-tw/configuration/memory-estimation/)；默認 4K 單路：`(43~50) + 1×122` → 約 165~172 MB。

通過 API 啟動高畫質錄製時，請顯式傳 `qn` 參數，詳見[錄製管理 API](/zh-tw/api/record/)。
