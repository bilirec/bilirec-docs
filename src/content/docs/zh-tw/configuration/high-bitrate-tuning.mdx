預設設定已覆蓋大多數場景：**錄 1080p 不用改**；錄 4K / 杜比時在 API 裡傳對應 `qn` 即可，程式會自動用更大塊的緩衝。

只有出現卡頓、記憶體不足、或 HLS 分片特別大時，再按下表微調。

## 高畫質自動切換

bilirec 為 **1080p** 與 **高畫質（2K/4K/杜比）** 各準備了一套緩衝。錄製時根據 `qn` 自動選用，無需在設定裡手動開「4K 模式」。

| 畫質 | 判定（`qn`） | 讀入塊 | 讀入排隊 | 寫盤塊 |
| ---- | ------------ | ------ | -------- | ------ |
| 1080p 及以下 | `< 20000` | 512 KB | 16 塊 | 512 KB |
| 2K / 4K / 杜比 | `≥ 20000` | 1 MB | 48 塊 | 1 MB |

環境變數對應關係：`READ_STREAM_BYTES_POOL_SIZE` / `*_HIGH`、`READ_STREAM_CHAN_BUFFER_SIZE` / `*_HIGH`、`LIVE_STREAM_WRITER_BYTES_POOL_SIZE` / `*_HIGH`。

| 要點 | 說明 |
| ---- | ---- |
| 按需佔用 | 只有真的在錄高畫質時才用 `_HIGH` 那套；不錄 4K 不會多佔記憶體 |
| HLS | 分片大小由 m3u8 決定；`qn ≥ 20000` 時讀入排隊深度仍用 `READ_STREAM_CHAN_BUFFER_SIZE_HIGH` |
| 寫盤排隊 | `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE` 不分檔，1080p 與 4K 共用 |

## 常見場景

| 場景 | 要不要改設定 | 建議 |
| ---- | ------------ | ---- |
| 1080p 原畫 | 否 | 保持預設 |
| 首次錄 4K / 杜比 | 通常否 | API 傳 `qn=20000`（或 `30000`）即可 |
| 4K 網路經常卡一下 | 是 | 加大 `READ_STREAM_CHAN_BUFFER_SIZE_HIGH`（預設 48） |
| 4K HLS 分片很大 | 是 | 主要加大 `LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH` |
| 4GB 設備多路 4K | 視情況 | 優先減小 `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE`（如 `48`）或 `MAX_CONCURRENT_RECORDINGS` |

## 推薦起點

三個 `*_HIGH` 預設值通常已夠用，下表供參考：

| 場景 | `READ_STREAM_BYTES_POOL_SIZE_HIGH` | `READ_STREAM_CHAN_BUFFER_SIZE_HIGH` | `LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH` | `LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE` |
| ---- | ---------------------------------- | ----------------------------------- | ----------------------------------------- | ------------------------------------- |
| 4K / 杜比 FLV 單路 | `1048576` (1 MB) | `48` | `1048576` (1 MB) | `64` |
| 4K / 杜比 HLS 單路 | 對 FLV 讀塊無影響 | `48` | `1048576` (1 MB) | `64` |
| SSD / NAS 單路極高碼率 | `1048576`～`2097152` | `48`～`64` | 與讀塊 `*_HIGH` 相同 | `64`～`128` |

<Aside type="caution">
調 4K 請改帶 `_HIGH` 後綴的變數。不要動不帶 `_HIGH` 的基礎變數——那是 1080p 用的，改大只會讓**所有**錄製都多佔記憶體。
</Aside>

## 示例（4K 預設仍不夠用時）

```bash
export READ_STREAM_BYTES_POOL_SIZE_HIGH=2097152       # 2 MB，與寫側保持一致
export READ_STREAM_CHAN_BUFFER_SIZE_HIGH=48
export LIVE_STREAM_WRITER_BYTES_POOL_SIZE_HIGH=2097152
export LIVE_STREAM_WRITER_CHAN_BUFFER_SIZE=64       # 多路 4K 可降到 48
```

**記憶體粗算（單路上限）：** 見 [記憶體佔用估算](/zh-tw/configuration/memory-estimation/)；預設 4K 單路約 165~172 MB。

通過 API 啟動高畫質錄製時，請顯式傳 `qn` 參數，詳見 [錄製管理 API](/zh-tw/api/record/)。
