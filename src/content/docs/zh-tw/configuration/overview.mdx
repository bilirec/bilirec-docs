所有設定通過**環境變數**設定。內容與 bilirec 倉庫 README 同步維護；完整說明以本站為文檔入口。

## 載入順序

1. 先嘗試讀取當前工作目錄的 **`.env.local`**（若存在）
2. 若不存在 `.env.local`，則讀取（或生成）可執行檔案同目錄的 **`.env`**
3. 在 **Docker 環境**（檢測到 `/.dockerenv`）下，跳過 `.env` 自動載入，僅使用容器環境變數
4. 你也可以直接設定系統環境變數，覆蓋 `.env.local` / `.env` 中的值

## 修改設定

**二進制模式：**

編輯 `.env.local` 或 `.env` 檔案，或直接設定環境變數：

**Linux / macOS：**

```bash
export MAX_CONCURRENT_RECORDINGS=5
./bilirec-linux-amd64
```

**Windows PowerShell：**

```powershell
$env:MAX_CONCURRENT_RECORDINGS="5"
.\bilirec-windows.exe
```

**Docker 模式：**

通過 `-e` 傳入環境變數：

```bash
docker run -e MAX_CONCURRENT_RECORDINGS=5 ...
```

## 重要說明

- `PUBLIC_BASE_URL` 為空或無效時，預簽名 URL 會回退為相對地址；CloudConvert 在該情況下會拒絕創建任務
- Web Push 的 VAPID key 在啟動時自動生成並寫入 `SECRET_DIR`，後續重啟會複用已有 key
