所有設定通過**環境變量**設置。bilirec 支持多種設定來源，按以下優先級加載：

## 加載順序

1. **`.env.local`** — 當前工作目錄下（若存在，最高優先級）
2. **`.env`** — 可執行檔案同目錄（首次運行自動生成模板）
3. **環境變量** — 覆蓋檔案中的值
4. **Docker 環境** — 檢測到 `/.dockerenv` 時跳過 `.env` 加載，僅使用環境變量

## 修改設定

**二進制模式：**

編輯 `.env.local` 或 `.env` 檔案，或直接設置環境變量：

```bash
export MAX_CONCURRENT_RECORDINGS=5
./bilirec-linux-amd64
```

**Docker 模式：**

通過 `-e` 傳入環境變量：

```bash
docker run -e MAX_CONCURRENT_RECORDINGS=5 ...
```

## 重要說明

- `PUBLIC_BASE_URL` 為空或無效時，預簽名 URL 會回退為相對地址；CloudConvert 在該情況下會拒絕創建任務
- Web Push 的 VAPID key 在啟動時自動生成並寫入 `SECRET_DIR`，後續重啟會複用已有 key
