---
title: Bilibili 登入
description: controller 模式下的 B 站掃碼登入 API
---

當 `BILIBILI_LOGIN_MODE=controller` 時，通過 `/auth/bilibili` 接口在服務運行後觸發掃碼登入。

## 查詢認證狀態

```http
GET /auth/bilibili/status
```

返回當前認證狀態（`idle` / `awaiting_qr` / `authenticating` / `authenticated` / `failed`）。

**已登入：**

```json
{
  "authenticated": true,
  "state": "authenticated",
  "account": {
    "mid": 123456789,
    "uname": "bilibili_user"
  }
}
```

**等待掃碼：**

```json
{
  "authenticated": false,
  "state": "awaiting_qr",
  "qr": {
    "url": "https://api.bilibili.com/x/web-interface/qrcode/generate?qrcode_key=abc123..."
  }
}
```

## 初始化二維碼登入

```http
POST /auth/bilibili/init
```

創建新的二維碼登入會話。支持帳號切換，即使已登入也可重新發起。

- 成功：`201 Created`
- `400` — 未啟用 controller 模式
- `409` — 已存在未完成的登入會話
- `500` — 獲取二維碼失敗

**成功響應示例：**

```json
{
  "qr": {
    "url": "https://api.bilibili.com/x/web-interface/qrcode/generate?qrcode_key=def456..."
  }
}
```

**`400 Bad Request` 示例：**

```json
{
  "error": "後端沒有啟用 controller 模式"
}
```

**`500 Internal Server Error` 示例：**

```json
{
  "error": "failed to get QR code: ..."
}
```

## 典型流程

1. `POST /login`（若啟用了 `USERNAME` / `PASSWORD`）
2. `POST /auth/bilibili/init` 獲取二維碼
3. 用戶掃碼後，輪詢 `GET /auth/bilibili/status` 直到 `state=authenticated`

```bash
curl -i -c cookies.txt -X POST http://127.0.0.1:8080/login \
  -H "Content-Type: application/json" \
  -d '{"user":"admin","pass":"changeme"}'

curl -i -b cookies.txt -X POST http://127.0.0.1:8080/auth/bilibili/init

curl -s -b cookies.txt http://127.0.0.1:8080/auth/bilibili/status
```

已登入用戶可隨時調用 `POST /auth/bilibili/init` 切換到新帳號。
