bilirec 內置 FRP 客戶端，無需公網 IP 即可在外網安全存取後端管理任務與檔案。

## 啟用 FRP

最簡單的方式——使用官方免費公共服務：

```bash
export FRP_ENABLED=true
```

僅設定 `FRP_ENABLED=true` 即可，程式會自動使用 bilirec.org 官方 FRP 服務。

若要通過 [app.bilirec.org](https://app.bilirec.org/) 登入並管理後端，還需同時開啟生產模式並設定帳號密碼（見下文）。

## 操作流程

1. **啟動後端**並確認 FRP 已啟用
2. **查看日誌**，找到公網存取地址。啟動成功後會顯示「內網穿透已建立」信息框，**遠端地址**一行即公網地址：

   ![啟動日誌中的公網地址](/images/frp/tunnel-log.png)

3. **打開** [app.bilirec.org](https://app.bilirec.org/)
4. 在登入介面的**伺服器地址**欄填入上一步的公網地址，再輸入 `USERNAME` 和 `PASSWORD` 登入：

   <img src="/images/frp/login-server-url.png" alt="在登入頁填入公網地址" width="400" style="max-width:100%;height:auto;display:block;margin:0 auto" />

<Aside type="caution">
暴露到公網後，**建議**設定 `USERNAME` 和 `PASSWORD`，否則 API 將完全公開存取。詳見[認證與安全](/zh-tw/configuration/auth-security/)。
</Aside>

## 通過 app.bilirec.org 登入

<Aside type="note">
以下說明**不只適用於 FRP**，也適用於你在 NAS、VPS 等**自架 HTTPS 後端**（例如 `https://api.example.com`、Nginx 反代後的域名）。只要管理頁在 [app.bilirec.org](https://app.bilirec.org/)，而後端在**另一個網址**，就可能遇到同樣的問題。
</Aside>

管理頁和後端地址（無論是 FRP 的 `https://abc123.tunnel.bilirec.org`，還是自架的 `https://api.example.com`）是**兩個不同的網站**。登入後，瀏覽器需要「記住」你已登入；預設設定下往往記不住，於是出現：

- 點了登入，介面顯示成功
- 剛進主介面幾秒，又彈出「會話已過期，請重新登入」
- 反覆登入仍一樣

部分瀏覽器的隱私設定較嚴，在**特定組合**下也會攔下登入狀態（見下文「根域名是否相同」）。

**推薦設定（通過 app.bilirec.org 連接遠端 HTTPS 後端時）：**

```bash
export PRODUCTION_MODE=true
export USERNAME=你的用戶名
export PASSWORD=你的密碼
export JWT_SECRET=一個固定的長隨機字符串
export FRONTEND_URL=https://app.bilirec.org
```

若使用 FRP，再加上 `export FRP_ENABLED=true`。改完後**重啟 bilirec**，在登入頁填入**當前**的後端地址再登入（FRP 每次重啟子域名可能變化）。

### 根域名是否相同

管理頁和後端是否屬於**同一個根域名**（例如都是 `bilirec.org` 旗下），決定了除 `PRODUCTION_MODE=true` 外，是否還要處理瀏覽器的第三方 Cookie 限制：

| 管理頁 | 後端（FRP 等） | 根域名 | 一般還需允許第三方 Cookie？ |
|--------|----------------|--------|---------------------------|
| [app.bilirec.org](https://app.bilirec.org/) | `https://xxx.tunnel.bilirec.org` | 同為 `bilirec.org` | **通常不需要** — 即使瀏覽器封鎖第三方 Cookie，登入也大多正常 |
| 自架前端，如 `https://bilirec.example.com` | `https://xxx.tunnel.bilirec.org` 或其它不同根域名的地址 | **不同**（如 `example.com` 與 `bilirec.org`） | **往往需要** — 瀏覽器會把後端的登入狀態當作「第三方」，預設可能攔下 |

**自架前端 + 不同根域名後端**時，除 `PRODUCTION_MODE=true` 外，還須把後端的 `FRONTEND_URL` 設為**你的前端 HTTPS 地址**（不能仍寫 `app.bilirec.org`）。若仍登入不上，再到瀏覽器裡**允許第三方 Cookie**，或將前端與後端域名加入例外：

| 瀏覽器 | 大致路徑 |
|--------|---------|
| Chrome / Edge | 設定 → 隱私和安全 → 第三方 Cookie → 選擇「允許第三方 Cookie」，或將**前端域名**與**後端域名**加入允許名單 |
| Firefox | 設定 → 隱私與安全 → 在「增強型跟蹤保護」中為前端域名關閉保護，或放寬 Cookie 限制 |
| Safari（iOS / macOS） | 設定 → Safari → 關閉「阻止跨站跟蹤」，或在前端站點允許 Cookie |

改完瀏覽器設定後，重新打開管理頁再登入一次。

本機開發（`http://localhost:8080`）不受此影響。詳見 [認證與安全 — 生產模式](/zh-tw/configuration/auth-security/#何時需要生產模式) 與 [常見問題](/zh-tw/guides/faq/#用-appbilirecorg-連遠端後端一直提示會話過期)。

## 自建 FRP 伺服器

若你有自己的 FRP 服務，可自定義設定：

```bash
export FRP_ENABLED=true
export FRP_SERVER=your-frp-ip:7000
export FRP_TOKEN=your-token
export FRP_BASE_DOMAIN=your-domain.com
```

啟動日誌會顯示 `FRP enabled in custom-selfhost mode`。

技術細節、協議約束與完整參數說明見 [FRP 設定](/zh-tw/configuration/frp/)。
