## 手動錄製

在 Web 介面中輸入直播間房間號，在確認頁選擇畫質、允許的流格式、是否僅錄製音頻與時長，點擊開始錄製。

<Aside type="note">
Web 介面中「畫質」為請求偏好，不可用時會盡量選用該直播支持的較高檔位。「流格式」可多選，作為嚴格允許清單：未選表示允許全部格式；選中後只會在所選格式內嘗試。「僅錄製音頻」對應 API 參數 `only_audio=true`。不傳 `qn` 時後端預設使用原畫（`10000`）。
</Aside>

也可通過 API 啟動：

```bash
curl -X POST "http://127.0.0.1:8080/record/12345/start?duration_minutes=30&stream_profile=http-flv&qn=150"
```

### 畫質碼（qn）

| 值 | 畫質 |
|----|------|
| `80` | 流暢 |
| `150` | 高清 |
| `250` | 超清 |
| `400` | 藍光 |
| `10000` | 原畫 |
| `20000` | 4K |
| `30000` | 杜比 |

### 僅錄製音頻（only_audio）

| 值 | 說明 |
|----|------|
| 不傳 / `false` | 錄製音影片 |
| `true` | 僅錄製音頻 |

### 流格式（stream_profile / stream_profiles）

嚴格允許清單。

- **開始錄製 API**：query 參數 `stream_profile`，可傳單個值，或多個值用英文逗號分隔。
- **房間設定 API**：JSON 字段 `stream_profiles`，為字符串數組。

| 值 | 說明 |
|----|------|
| 不傳 / 空 | 允許全部格式；同畫質下按 HTTP-FLV → HLS-fMP4 → HLS-TS 排序嘗試 |
| `http-flv` | 僅 HTTP-FLV（PK/分辨率切換時自動分段） |
| `hls-ts` | 僅 HLS-TS（不執行檔案輪轉） |
| `hls-fmp4` | 僅 HLS-fMP4（預設將 init 與媒體分片寫入同一檔案；僅 init 內容變更時分段） |

多選時只會在所選格式內嘗試；同畫質下仍按 FLV → fMP4 → TS 排序。想盡量錄到下播結尾時可選 HLS；要更貼近直播邊緣時可選 FLV。

### 錄製時長（duration_minutes）

| 值 | 說明 |
|----|------|
| 不傳 / `0` | 使用系統預設（`MAX_RECORDING_HOURS`） |
| `-1` | 無限錄製 |
| 正整數 | N 分鐘後自動停止 |

## 訂閱房間

訂閱直播間後，可設定開播自動錄製與開播通知：

1. 在 Web 介面添加房間訂閱
2. 設定自動錄製與通知選項

通過 API：

```bash
# 訂閱房間
curl -X POST http://127.0.0.1:8080/room/12345

# 設定自動錄製（可同時設定畫質與允許的流格式）
curl -X PUT http://127.0.0.1:8080/room/12345/config \
  -H "Content-Type: application/json" \
  -d '{"auto_record":true,"notify":true,"record_duration_minutes":120,"qn":150,"stream_profiles":["hls-fmp4","hls-ts"]}'
```

為房間開啟 **開播通知** 或 **自動錄製** 後，後台會定期檢查該房間的直播狀態。服務啟動時先檢查所有相關房間一次，之後將房間分成若干組、分組輪流檢查，避免訂閱較多時 CPU 瞬間飆高。檢測到新一場直播時：若開啟了通知則推送消息；若開啟了自動錄製則自動開始錄製（沿用房間設定中的畫質、流格式允許清單等）。分組與檢查週期預設隨訂閱規模自動調整，詳見 [錄製設定](/zh-tw/configuration/recording/#訂閱房間狀態檢查)。

## 自動錄製

啟用 `auto_record: true` 後，檢測到開播時會自動開始錄製。

`record_duration_minutes` 控制自動錄製的時長上限（僅在 `auto_record: true` 時生效）。房間設定中的 `qn`、`stream_profiles`、`only_audio` 會作為自動錄製參數一併生效。

<Aside type="note">
手動觸發的錄製不受房間 `record_duration_minutes` 設定影響，需在調用 `/record/:roomID/start` 時自行傳入 `duration_minutes` 參數。
</Aside>

## 停止錄製

```bash
curl -X POST http://127.0.0.1:8080/record/12345/stop
```

完整 API 參考見 [錄製管理 API](/zh-tw/api/record/)。

<Aside type="tip">
**常見問題：** 直播中斷後出現多個檔案是正常分段；CPU 飆高可能與訂閱檢查有關。詳見 [常見問題](/zh-tw/guides/faq/#後端錄製) 與 [性能與容量規劃](/zh-tw/guides/faq/#性能與容量規劃)。
</Aside>
