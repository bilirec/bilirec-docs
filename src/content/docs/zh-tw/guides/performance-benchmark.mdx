本文記錄 bilirec 在低配設備上的實測表現，便於你評估是否適合自己的部署環境。

## 測試環境

- 設備：樹莓派 5B 16GB
- 容器：Docker（1 GB 內存限制）
- 場景：默認設定，每路直播 1080p，同時錄製多個直播間
- 指標：CPU 峰值佔用、內存峰值佔用

<Aside type="note">
以下數據僅供參考（實測快照，改設定或版本後會有偏差）。下表為默認設定下的**常見實測峰值**；按環境變量規劃容器上限請用 [內存佔用估算](/zh-tw/configuration/memory-estimation/)（偏保守，通常高於本表）。
</Aside>

## 數據彙總

| 併發路數 | CPU 峰值佔用 | 內存峰值佔用 |
|---------|------------|-------------|
| 初始閒置 | ~0.0% | ~7MB |
| 1 路併發 | ~1.8% | ~72MB |
| 2 路併發 | ~2.7% | ~117MB |
| 3 路併發 | ~3.1% | ~141MB |
| 4 路併發 | ~3.8% | ~165MB |
| 5 路併發 | ~4.6% | ~191MB |
| 恢復閒置 | ~0.0% | ~54MB |

<Aside type="note">
bilirec 使用了大量內存池以降低 GC 壓力，停止錄製後部分內存會保留複用（見上表「恢復閒置」），以減輕 GC 壓力。內存佔用與緩衝設定、錄製路數相關；**真實佔用請按 [內存佔用估算](/zh-tw/configuration/memory-estimation/) 計算**，勿直接套用上表數字。
</Aside>
