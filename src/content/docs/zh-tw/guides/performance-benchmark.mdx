本文記錄 bilirec 在低配設備上的實測表現，便於你對照自己的環境。規劃容器記憶體上限請用 [記憶體佔用估算](/zh-tw/configuration/memory-estimation/)，不要直接套用下表數字。

## 測試環境

- 設備：樹莓派 5B 16GB
- 容器：Docker（1 GB 記憶體限制）
- 場景：預設設定，每路直播 1080p，同時錄製多個直播間
- 指標：CPU 峰值佔用；記憶體峰值 = 容器監控顯示的記憶體用量

<Aside type="note">
下表為實測快照，改設定或版本後會有偏差。
</Aside>

## 數據彙總

| 並發路數 | CPU 峰值佔用 | 記憶體峰值佔用 |
|---------|------------|-------------|
| 初始閒置 | ~0.0% | ~7 MB |
| 1 路並發 | ~2.1% | ~65 MB |
| 2 路並發 | ~2.6% | ~86 MB |
| 3 路並發 | ~3.6% | ~103 MB |
| 4 路並發 | ~4.1% | ~144 MB |
| 5 路並發 | ~4.9% | ~173 MB |
| 恢復閒置 | ~0.0% | ~54 MB |

<Aside type="note">
停錄後記憶體會比剛啟動時高一些，是因為程式會保留一部分緩衝供下次錄製複用（見上表「恢復閒置」），屬於正常現象。
</Aside>

<Aside type="tip">
監控數字偏高、怎麼評估自己能錄幾路等問題 — 見 [常見問題](/zh-tw/guides/faq/#性能與容量規劃)。
</Aside>
