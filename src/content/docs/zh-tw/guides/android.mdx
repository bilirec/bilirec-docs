## 安裝

從 [bilirec-mobile Releases](https://github.com/bilirec/bilirec-mobile/releases) 下載並安裝 APK。

推薦使用 **Android 14（API 34）** 或更高版本。bilirec-mobile 理論上最低支持 **Android 10（API 29）**；更低版本未經完整驗證，可能無法安裝或無法穩定運行前臺錄製服務。

## 控制中心

App 主介面為 **Bilirec 服務控制中心**，核心元素如下：

| 區域 | 說明 |
|------|------|
| 頂部標題欄 | 顯示「Bilirec 服務控制中心」；右上角可切換 **繁 / 簡** 介面語言 |
| 中央電源按鈕 | 啟動或停止 bilirec 系統服務（前臺服務 + 內嵌後端） |
| 狀態行 | 顯示當前服務狀態（未啟動 / 啟動中 / 運行中 / 停止中等） |
| 服務運行後 | 出現 **打開錄製管理程式**、**檢查系統服務連接** 兩個操作按鈕 |
| 底部設定入口 | **打開服務啟動前設定** 按鈕；**僅在服務未運行且未處於啟停過程中時可點擊** |

## 首次使用與權限

首次打開 App 時，通常會依次遇到以下系統權限與限制：

1. **電池優化豁免** — 彈窗提示將 Bilirec 設為「不受電池限制」。未完成前無法打開設定頁，也無法啟動服務。點擊「前往設定」按系統指引操作後返回 App。
2. **通知權限** — 點擊電源按鈕啟動服務時，若尚未授予通知權限，系統會請求授權。通知用於常駐前臺服務；拒絕則無法啟動。
3. **（可選）輸出路徑** — 若要把錄製檔案保存到 microSD 卡或指定檔案夾，可在啟動服務前進入設定 → **儲存策略** → **更改路徑** 選擇目錄。

## 使用流程

1. （推薦）點擊 **打開服務啟動前設定**，按需設定輸出路徑、microSD 卡磨損保護、錄製策略等（見下文）
2. 點擊中央 **啟動** 按鈕，等待狀態變為「Bilirec 系統服務運行中」
3. 點擊 **打開錄製管理程式**，跳轉到 [app.bilirec.org](https://app.bilirec.org/)（若已安裝 PWA 則優先打開 PWA）
4. 在 Web 介面點擊「登入」完成 B 站掃碼認證，即可訂閱房間或開始錄製
5. 錄製結束後可點擊 **停止** 關閉系統服務；也可在通知欄點擊「停止服務」

<Tabs>
<TabItem label="檢查連接">

若不確定後端是否就緒，可點擊 **檢查系統服務連接**。App 會請求 `http://127.0.0.1:8080/` 並提示連接正常、響應異常或無響應。

</TabItem>
<TabItem label="前臺通知">

服務運行期間，通知欄會常駐顯示 CPU / 記憶體佔用。正在錄製時，通知正文前會標註「錄製中」。若開啟了 **激進防休眠模式** 且已生效，通知標題會出現 ⚡（Wi‑Fi 鎖定時還帶 🛜 或 🌐 圖標）。

</TabItem>
</Tabs>

<Aside type="note">
Android 後端預設監聽 `http://127.0.0.1:8080`（登入頁填 `localhost` 或 `127.0.0.1` 均可），且預設不啟用帳號密碼認證。後端在 App 進程內運行，僅本機可存取。
</Aside>

### 一般設定

| App 選項 | 作用 | 對應設定 |
|---------|------|---------|
| **本地通知模式** | 在無法使用 Web Push 時（如部分大陸網路），改由 App 通過 SSE 拉取開播/錄製事件並彈出**本地通知** | 啟動時自動生成 `NOTIFY_SSE_TOKEN` |
| **激進防休眠模式** | 關屏閒置 1～2 小時後若出現錄製斷斷續續，可嘗試開啟；錄製期間會啟用網路鎖降低休眠影響 | App 側開關（非環境變數） |

<Aside type="caution">
開啟 **本地通知模式** 後，點擊通知**無法**直接跳轉到錄製管理程式，需手動打開 App 或 PWA。詳見 [通知設定](/zh-tw/configuration/notifications/)。
</Aside>

開啟 **激進防休眠模式** 會增加耗電，長時間錄製建議接電源。通知標題出現 ⚡ 表示防休眠已生效。

### 儲存策略

| App 選項 | 作用 | 對應設定 |
|---------|------|---------|
| **輸出路徑** | 選擇錄製檔案保存目錄；未設定時使用 App 私有目錄下的預設 `records` | `OUTPUT_DIR`（獨立儲存，非 managed env） |
| **microSD 卡磨損保護** | 輪流寫入錄製檔案，降低 microSD 同時寫入峰值；寫入節奏隨「同時錄製上限」自動調節 | `SEQUENTIAL_WRITE` + `LIVE_STREAM_WRITER_FLUSH_PERIOD_SECS` |

若輸出路徑選擇 **microSD 卡**，建議開啟 **microSD 卡磨損保護**。

### 錄製策略

以下選項通過滑塊或分段按鈕調整，修改後立即保存，下次啟動服務時生效：

| App 選項 | 環境變數 | 可選範圍 / 說明 |
|---------|---------|----------------|
| **單次錄製時長上限** | `MAX_RECORDING_HOURS` | `0`（不限）～ `12` 小時；滑塊顯示「不限」即 `0` |
| **啟動前最低可用空間** | `MIN_DISK_SPACE_BYTES` | `2` / `5` / `10` GB |
| **下線後服務重試時長** | `MAX_RETRY_MINUTES` | `5`～`30` 分鐘（步進 5） |
| **重試成功後的時長計算** | `RECORDING_RECOVERY_DURATION` | **保留**（`preserve`）或 **重置**（`reset`）；斷流後若重試成功並恢復錄製，可選擇沿用開始錄製時的總時長，或從重試成功時重新計算錄製時長 |
| **同時錄製上限** | `MAX_CONCURRENT_RECORDINGS` | `3`～`6` 路；提高會明顯增加發熱與耗電 |

<Aside type="note">
**啟動前最低可用空間** 滑塊預設顯示 5 GB，但僅在你拖動並鬆手後才會寫入環境變數。若從未調整過，後端仍使用 [Android 預設設定](/zh-tw/configuration/android-defaults/) 中的 `2 GB` 門檻。
</Aside>

### 轉換策略

需先開啟 **錄完自動轉為 MP4**，其餘轉檔相關選項才會可用：

| App 選項 | 環境變數 | 說明 |
|---------|---------|------|
| **錄完自動轉為 MP4** | `CONVERT_TO_MP4` | 使用內嵌 FFmpeg 在錄製完成後轉 MP4；RAM 少於 8 GB 建議關閉 |
| **轉檔後刪除原始檔案** | `DELETE_SOURCE_AFTER_CONVERT` | 轉碼成功後再異步刪除源檔案；刪除失敗會重試，仍失敗則重命名為帶「轉換完成可刪除」標記的檔案 |
| **錄製中允許轉換** | `FFMPEG_ALLOW_DURING_RECORDING` | 邊錄邊轉，增加讀寫負載與發熱 |
| **允許轉檔的「最高錄製路數」** | `FFMPEG_ALLOW_DURING_RECORDING_MAX_ACTIVE_RECORDINGS` | `0`（不限）～`5` 路；超過時暫停後台轉檔以優先保證錄製 |

轉檔行為詳見 [轉換設定](/zh-tw/configuration/conversion/)。

### 開發者選項

面向進階用戶，一般錄製無需修改：

| App 選項 | 說明 |
|---------|------|
| **環境參數設定** | 手動添加 `Key` / `Value` 環境變數；**優先級高於**上方各策略開關，覆蓋同名字段 |
| **啟動日誌** | 合併導出 `bootstrap.log` 分片，便於反饋啟動或崩潰問題 |

<Aside type="danger">
建議避免隨意添加不熟悉的環境變數。錯誤設定可能導致發熱、耗電增加、後台被殺或錄製不穩定。
</Aside>

完整 App 設定與環境變數對照見上文各區塊說明。

## 預設行為

Android 模式下，bilirec 後端會先注入一組移動端預設值，再疊加 App 設定中的環境變數：

- `BILIBILI_LOGIN_MODE=controller` — 通過 Web UI 掃碼登入
- `FRP_ENABLED=false` — 手機無需內網穿透
- `SILENT_ACCESS_LOG=true` — 減少日誌輸出
- 更保守的 I/O / 記憶體參數（如較小寫入緩衝、預設關閉 `SEQUENTIAL_WRITE`）

完整預設值與伺服器版差異見 [Android 預設設定](/zh-tw/configuration/android-defaults/)。

## 國產手機（華為 / ColorOS 等）

華為、OPPO（ColorOS）、小米（HyperOS）、vivo（OriginOS）等大陸國產系統的後台限制比一般 Android 更嚴。除 App 內的電池優化豁免外，可能還需在系統設定中逐步調整（多任務鎖定、自啟動、省電放行等）。**按順序逐步嘗試即可，問題解決後不必全部做完。**

若使用移動數據時仍頻繁超時，可暫時改用 Wi‑Fi。完整圖文步驟見 [國產手機後台設定](/zh-tw/guides/android-mainland/)。
