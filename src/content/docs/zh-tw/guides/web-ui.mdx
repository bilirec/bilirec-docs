bilirec 的 Web 管理介面由 [bilirec-web](https://github.com/bilirec/bilirec-web) 提供，線上地址為 [app.bilirec.org](https://app.bilirec.org/)。它是一個通用前端，可連接任何 bilirec 後端實例。

## 功能概覽

| 功能 | 說明 |
|------|------|
| 錄製管理 | 手動開始/停止錄製，查看 `recording` / `recovering` / `idle` 狀態與統計 |
| 直播訂閱 | 訂閱房間、設定自動錄製與開播通知 |
| 網頁推送 | 通過 Service Worker 接收開播等 Web Push 通知 |
| 檔案瀏覽 | 瀏覽錄製檔案，支持下載與 MP4 預覽 |
| 檔案共享 | 生成帶過期時間的分享鏈接 |
| 格式轉換 | 提交與管理 FLV/TS/FMP4 → MP4 轉換任務 |
| 磁碟監測 | 實時磁碟佔用與分項統計 |
| 角色控制 | 管理員 / 查看者（查看者為只讀介面） |
| 深色模式 | 淺色/深色主題，支持跟隨系統 |
| 多語言 | 簡體中文 / 繁體中文 |
| 多伺服器 | 登入頁可設定後端地址，連接不同 bilirec 實例 |
| PWA | 可安裝到桌面或手機主屏幕，獲得接近原生 App 的體驗 |

<Aside type="tip">
bilirec-web 需配合正在運行的 [bilirec](https://github.com/bilirec/bilirec) 後端使用。也可[自行部署前端](/zh-tw/development/web/)以使用自定義域名；自架 HTTPS 前端與官方頁一樣，不能直連局域網 HTTP 後端，須本機 `localhost` / `127.0.0.1`、後端 HTTPS，或前後端同站反代。
</Aside>

## 存取 Web 介面

1. 確保 bilirec 後端已啟動（預設端口為 `8080`，`HOST` 預設為空值即監聽所有網卡）
2. 打開 [app.bilirec.org](https://app.bilirec.org/)

<Tabs>
<TabItem label="桌面端">

![桌面端登入介面](/images/web-ui/desktop-login.png)

</TabItem>
<TabItem label="移動端">

![移動端登入介面](/images/web-ui/mobile-login.png)

</TabItem>
</Tabs>

3. 在登入介面填入後端地址（本地使用可留空或填 `http://localhost:8080`）
4. 若設定了 `USERNAME` 和 `PASSWORD`，輸入帳號密碼登入；未設定則直接進入

進入主介面後，點擊右上角頭像（移動端為設定），在菜單中完成 B 站掃碼登入。

## 連接自架後端

[app.bilirec.org](https://app.bilirec.org/) 通過 **HTTPS** 提供，而多數自架 bilirec 後端預設只提供 **HTTP**。瀏覽器出於混合內容（Mixed Content）安全策略，**禁止 HTTPS 頁面請求 HTTP 的局域網地址**（如 `http://192.168.x.x:8080`），**僅允許**連接本機：

- `http://localhost:8080`
- `http://127.0.0.1:8080`

| 後端位置 | 能否用 app.bilirec.org 連接 |
|---------|---------------------------|
| 本機（bilirec-mobile、PC 本機後端） | 可以，填 `localhost` 或 `127.0.0.1` |
| 局域網另一台設備上的 HTTP 後端 | **不能** — HTTPS 前端無法直連 HTTP 局域網地址 |
| 公網 HTTPS（FRP、自架域名、反向代理等） | 可以，填 `https://你的後端地址`；需 `PRODUCTION_MODE=true`，見 [常見問題](/zh-tw/guides/faq/#用-appbilirecorg-連遠端後端一直提示會話過期)。官方頁連 `*.tunnel.bilirec.org` 同屬 `bilirec.org`，一般不必為登入允許第三方 Cookie |

遠端存取局域網內的 HTTP 後端，可考慮 [FRP](/zh-tw/guides/frp/) 等提供 **HTTPS** 的方案。自架 HTTPS 後端通過 app.bilirec.org 登入時，建議開啟生產模式。[自架前端](/zh-tw/development/web/)且與後端**根域名不同**時，可能還需允許第三方 Cookie，見 [FRP — 根域名是否相同](/zh-tw/guides/frp/#根域名是否相同)。

## 安裝 PWA

bilirec-web 支持 PWA（Progressive Web App），可安裝到桌面或手機主屏幕，獲得接近原生 App 的體驗。安裝後會記住上次連接的後端地址。

<Tabs>
<TabItem label="Android Chrome">

若瀏覽器檢測到可安裝，頁面頂部會出現安裝橫幅，點擊 **安裝** 即可：

<img src="/images/web-ui/pwa-install-banner.png" alt="Android Chrome PWA 安裝橫幅" width="400" style="max-width:100%;height:auto;display:block;margin:0 auto" />

也可通過菜單 **⋮**，選擇 **新增至主屏幕**（部分機型顯示為「安裝應用」）：

<img src="/images/web-ui/pwa-menu-add-home.png" alt="Chrome 菜單「新增至主屏幕」" width="400" style="max-width:100%;height:auto;display:block;margin:0 auto" />

在確認對話框中點擊 **安裝**：

<img src="/images/web-ui/pwa-install-dialog.png" alt="PWA 安裝確認對話框" width="400" style="max-width:100%;height:auto;display:block;margin:0 auto" />

</TabItem>
<TabItem label="桌面瀏覽器（Chrome / Edge）">

打開 [app.bilirec.org](https://app.bilirec.org/) 後，點擊地址欄右側的 **安裝** 圖標：

<img src="/images/web-ui/pwa-desktop-login.png" alt="桌面 Chrome 登入頁與地址欄安裝按鈕" style="max-width:100%;height:auto;display:block;margin:0 auto" />

在「安裝應用程式」對話框中點擊 **安裝**：

<img src="/images/web-ui/pwa-desktop-install-dialog.png" alt="桌面 Chrome PWA 安裝確認對話框" style="max-width:100%;height:auto;display:block;margin:0 auto" />

若地址欄沒有安裝圖標，可打開菜單 **⋮**，選擇 **安裝 Bilibili 錄製管理**（或「安裝 bilirec」）。

</TabItem>
<TabItem label="iOS Safari">

點 **分享**，再選 **添加到主屏幕**。

</TabItem>
</Tabs>
